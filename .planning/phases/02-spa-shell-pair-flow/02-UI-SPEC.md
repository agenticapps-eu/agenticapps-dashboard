# Phase 2 — UI Design Contract

> Visual and interaction contract for the SPA shell + pair flow. Generated by gsd-ui-researcher, verified by gsd-ui-checker. Anti-AI-slop discipline is non-negotiable; every dimension below is the answer to "would this score ≥ 90 on `impeccable:critique`?" — Phase 6 measures that, Phase 2 sets the baseline.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (intentional — no shadcn registry, no Radix wrappers, no headless-UI lib) |
| Preset | not applicable |
| Component library | hand-rolled primitives in `packages/spa/src/components/`. Tailwind v4 utility classes only; no UI lib. Reasoning: the dashboard's own UI must pass `impeccable:critique` ≥ 90 — every imported component widens the audit surface. |
| Icon library | `lucide-react` (catalog: `^1.14.0`, already pinned). Allowed icons in Phase 2: `Sun`, `Moon`, `Monitor` (system theme), `Copy`, `Check` (copy success), `X` (banner dismiss), `AlertTriangle` (banner + drift), `RefreshCw` (drift reload), `Settings`, `ChevronDown` (disclosure). No emoji as functional UI affordance — anywhere. |
| Font | System sans default stack: `ui-sans-serif, system-ui, -apple-system, "Segoe UI", Roboto, Inter, sans-serif`. Mono: `ui-monospace, SFMono-Regular, "JetBrains Mono", Menlo, Consolas, monospace`. Reasoning: zero web-font network requests preserves the no-third-party-JS pillar AND the "Nothing leaves your machine" value prop verbatim — even font CDNs are an exfiltration surface. |
| Theme strategy | `dark` class on `<html>`. Tailwind v4 `@custom-variant dark (&:where(.dark, .dark *));` in `global.css`. CSS variables drive both themes from a single component tree (D-02). |
| Default theme | dark (D-02). |
| Theme persistence | `localStorage["agentic-dashboard:theme"]` ∈ `"dark" \| "light" \| "system"` (D-03). |

## Spacing Scale

**Base: 4px.** All spacing values are multiples of 4. Use 8px increments above `md` to match Tailwind v4 defaults; the 4px base permits `xs` for icon-tight contexts (lucide-react default optical alignment).

| Token | Value | Tailwind class | Usage |
|-------|-------|----------------|-------|
| 0  | 0px   | `p-0` / `gap-0` | Reset only |
| xs | 4px   | `p-1` / `gap-1` | Icon ↔ text gap inside a button or chip; tight inline padding inside `<CodeBlock>` row chrome |
| sm | 8px   | `p-2` / `gap-2` | Form field internal padding; banner button group internal gap; numbered-step bullet ↔ text gap |
| md | 16px  | `p-4` / `gap-4` | Default block padding (cards, banner content, panel state body); paragraph-to-paragraph stack |
| lg | 24px  | `p-6` / `gap-6` | Section padding; OnboardingHero step-to-step vertical stack; ManualPairForm field-to-field stack |
| xl | 32px  | `p-8` / `gap-8` | Layout-region gaps (header → main, hero → steps); modal-like card padding |
| 2xl | 48px | `p-12` / `gap-12` | Page-level top padding on `/onboarding` (hero breathing room); major section breaks |
| 3xl | 64px | `p-16` / `gap-16` | Reserved for Phase 3+ multi-project home; not used in Phase 2 |

**Touch / hit-target exception:** Theme chip + banner [×] buttons must be ≥ 32×32px hit area even when icon is 16px (achieve via `p-2` on a 16px icon → 32px box). WCAG 2.2 AA recommends 24×24, 32×32 is comfortable on touch laptops. **No 44×44 carve-out** — that would be mobile-only sizing, and this dashboard is desktop-first.

**Layout container:** `max-w-3xl` (768px) for `/onboarding` and `/settings`. `max-w-4xl` (896px) for the eventual home page (Phase 3); Phase 2 root `/` placeholder uses `max-w-3xl` to match. Page horizontal padding: `px-6` (24px) on mobile, `px-8` (32px) ≥ md breakpoint.

Exceptions: focus ring (`outline-offset-2`) — that's an outline, not a margin/padding token. Permitted.

## Typography

**Two weights only: 400 (regular) and 600 (semibold).** Hierarchy via size + weight + spacing, never color alone.

| Role | Size | Weight | Line Height | Tailwind | Usage |
|------|------|--------|-------------|----------|-------|
| Body | 16px | 400 | 1.5 (24px) | `text-base font-normal leading-relaxed` (use `leading-6` for exact 24px) | Default paragraph copy in onboarding aside, panel descriptions, form helper text. |
| Label | 14px | 600 | 1.4 (≈20px) | `text-sm font-semibold leading-tight` | Form field labels, button labels, table headers, banner CTA. |
| Heading | 20px | 600 | 1.3 (26px) | `text-xl font-semibold leading-snug` | Panel headings (e.g. "Schema drift detected", "Daemon not running"); `/settings` section titles. |
| Display | 28px | 600 | 1.2 (≈34px) | `text-3xl font-semibold leading-tight tracking-tight` | OnboardingHero headline ("One local daemon. Every device.") only. |
| Caption | 12px | 400 | 1.4 (≈17px) | `text-xs font-normal leading-snug` | Footer build-stamp, "matched: …" diagnostic lines, tertiary disclosures. **Use sparingly** — overuse signals dense data, which Phase 2 doesn't have. |
| Code | 13px | 400 | 1.5 (≈20px) | `font-mono text-[13px] leading-relaxed` | Inside `<CodeBlock>`; `<SchemaDriftState>` field-path/expected/got values. |

**Verbatim values for the executor:**

```css
/* In global.css @theme block — referenced by Tailwind utilities. */
--text-body: 1rem;        /* 16px */
--text-label: 0.875rem;   /* 14px */
--text-heading: 1.25rem;  /* 20px */
--text-display: 1.75rem;  /* 28px */
--text-caption: 0.75rem;  /* 12px */
--text-code: 0.8125rem;   /* 13px */

--leading-body: 1.5;
--leading-label: 1.4;
--leading-heading: 1.3;
--leading-display: 1.2;
```

**Rules:**
- No font sizes outside this table. If the executor reaches for a 5th size, the spec is wrong — fix the spec, don't add a size.
- No font weight other than 400 or 600. **No 500.** No 700. No italic for hierarchy (italic is for emphasis-in-prose only; Phase 2 has no prose that needs it).
- `tracking-tight` (-0.025em) only on Display (28px). Body text uses default tracking.
- Numerals in `<CodeBlock>` are mono regardless of context.

## Color

**60 / 30 / 10 split.**
- 60% dominant surface: page background.
- 30% secondary: panel cards, banner, code blocks, header bar, form inputs.
- 10% accent: focus rings, primary CTA hover, theme-active indicator, copy-success flash, drift-state heading underline. Reserved-for list is enumerated below — never apply accent to "all interactive elements".

CSS variables drive both themes from a single class. Variables in `:root` (light) and `.dark` (dark). Defaults: `.dark` is applied to `<html>` on first paint per D-02.

**Accent reserved for (exhaustive list — Phase 2):**
1. Focus rings on every interactive element (buttons, links, form fields, theme chip, disclosure toggle).
2. Primary-button fill: the **single** primary CTA on `/settings` (the "Save & connect" button on `ManualPairForm`). Onboarding has no primary CTA — the steps ARE the action.
3. The "Re-pair" button label inside `<RepairBanner>` (text-only accent — banner background is `--danger-surface`, not accent).
4. Active-state indicator on the theme tri-state radio in `/settings` (a 2px left border on the selected option's row).
5. Hyperlink underline on the "Why local-only →" disclosure trigger and any `<a>` in onboarding aside.
6. Copy-success transient flash on `<CodeBlock>` (the icon swaps to `Check` and the icon color flashes `--success`, NOT accent — accent is reserved for resting/hover affordances; success uses success). _Correction for clarity: copy-success uses `--success`, not `--accent`. Accent stays cool/blue; success stays green._

**Accent NEVER applied to:**
- Plain text body or headings (color hierarchy uses text/text-muted only).
- Card borders (use `--border`).
- Hover states on non-CTA elements (use `--surface-elevated`).
- Banner [×] dismiss icon (use `--text-muted`).
- Numbered step bullets in OnboardingHero (use `--text` on `--surface-elevated` chip — the hierarchy is shape + size, not color).

**Destructive:** No destructive actions in Phase 2. (Token rotation, project unregister, registry clear all live in CLI / Phase 3+.) `--danger` in Phase 2 is *signal* color, not *action* color — used on the banner left rail and drift heading to communicate "something went wrong", not on a "Delete" button. If a destructive button materializes during execution, surface it for spec review — it doesn't belong in Phase 2.

## Component Contracts

Eleven components ship in Phase 2. For each: props (TypeScript signature shape), states, exact tokens, exact spacing.

### `<AppShell>`

**Purpose:** Root layout. Wraps `<Outlet />` from TanStack Router with a header bar + main column + persistent banner mount-point.

**Props:** None. Reads pairing + repair state from context.

**Layout:**
- `<html class="dark">` (or no class for light, `dark` is the default applied at first paint by `useTheme`).
- `<body>` → `<div id="root">` → `<AppShell>` → `<header>` (sticky top, 56px tall) + `<main>` (flex-1, max-w-3xl mx-auto, px-6 / md:px-8, py-8) + `<RepairBanner>` (mounted once, slotted into a fixed top container BELOW the header so dismissing it doesn't shift the header).

**Spacing:**
- Header internal padding: `px-6 py-3` (24/12).
- Header-to-banner gap: 0 (banner sits flush below header when present).
- Banner-to-main gap: `pt-8` (32px) on `<main>` when banner is mounted; same when absent (no layout jump on mount/unmount; reserve the space via CSS grid template OR don't reserve — the executor picks one and the checker verifies no CLS jank).

**Tokens:** `bg-[--bg]` on body; `bg-[--surface]` + `border-b border-[--border]` on header.

### `<Header>`

**Purpose:** Persistent product chrome. Phase 2 minimal: brand label left, theme chip + settings icon right.

**Props:** None.

**Composition (left → right):**
- `<a href="/">AgenticApps Dashboard</a>` — Label-sized text, `font-semibold`, `text-[--text]`. No logo image (no asset pipeline cost).
- (spacer `flex-1`)
- `<ThemeChip />` — 32×32px button.
- `<a href="/settings"><Settings /></a>` — 32×32px icon button (lucide `Settings`, 16px). Aria-label "Settings".

**Spacing:** Items separated by `gap-2` (8px). Right-edge buttons use `p-2` to make 16px icons into 32px hit targets.

**States:**
- Resting: `text-[--text-muted]` on icons.
- Hover: `text-[--text]` + `bg-[--surface-elevated]` on icon button.
- Focus-visible: 2px ring `--ring` + 2px offset `--bg`.

**Active route awareness (Phase 2):** none yet (no nav items beyond settings icon, which is always-visible). Phase 3 brings the project switcher + active-route highlight.

### `<ThemeChip>`

**Purpose:** Cycle dark → light → system → dark. Always visible in header.

**Props:** None (reads `useTheme()` hook).

**Visual:** Single 32×32px button. Icon swaps based on current value:
- `"dark"` → `<Moon size={16} />`
- `"light"` → `<Sun size={16} />`
- `"system"` → `<Monitor size={16} />`

Aria-label is dynamic and reflects the **next** state, e.g. when current is dark: `aria-label="Switch theme (current: dark; next: light)"`. Tooltip via `title` attribute mirrors this.

**States:**
- Resting: `text-[--text-muted]`, transparent background.
- Hover: `text-[--text]` + `bg-[--surface-elevated]`.
- Focus-visible: ring (see ring spec).
- Active (mousedown): `bg-[--surface-elevated]` slightly darker (`brightness-95`).

**Interaction:**
- Click → cycles. Persists via `setItem("agentic-dashboard:theme", next)` and toggles `<html class="dark">` accordingly.
- Keyboard: `Enter` and `Space` both activate (default button behavior). No arrow-key cycling on the chip — the `/settings` tri-state radio is the keyboard-navigable form.

### `<RepairBanner>` (D-06)

**Purpose:** Surface 401 from daemon as a **persistent dismissible top banner**. Does NOT block the page. Stale data stays visible underneath.

**Props:**
```ts
type RepairBannerProps = {
  /** undefined when no 401 has occurred since last successful /health */
  needsRepair: boolean
  /** persists per-session (clears on full reload) */
  onDismiss: () => void
  /** navigates to /onboarding with state.step = 'pair' */
  onRepair: () => void
}
```

**States:**
- Hidden: `needsRepair === false`.
- Visible: `needsRepair === true && !sessionDismissed`.
- Dismissed (this session): `needsRepair === true && sessionDismissed === true` — banner is invisible until a NEW 401 arrives (resets the dismiss flag).
- Cleared: a successful 200 from `/health` flips `needsRepair` to `false`; banner unmounts.

**Visual (ASCII mockup, dark theme):**

```
+------------------------------------------------------------+
|  [!] Agent token rejected.    [ Re-pair ]              [×] |
+------------------------------------------------------------+
```

Tokens & spacing:
- Container: `bg-[--danger-surface]` (red-500 @ 8% on dark) + 1px border `--danger` only on the LEFT EDGE (`border-l-2`); top/right/bottom borders use `--border`. Height: 48px (`py-3`, body line-height accommodates 16px text). Width: 100% of viewport, sits below the 56px header.
- Inner: `flex items-center gap-3 px-6` (24px sides; 12px gap).
- Icon: `<AlertTriangle size={16} />` colored `text-[--danger]`. The square icon takes 16×16; container around icon uses `flex items-center` only (no padding ring).
- Body label: `<span>Agent token rejected.</span>` — `text-[--text]`, Label size (14px / 600). Mid-sentence period; no exclamation point — no marketing voice in error UI.
- "Re-pair" button: `bg-transparent border border-[--accent] text-[--accent] hover:bg-[--accent] hover:text-[--accent-fg]`. Padding `px-3 py-1` (12/4 → 24px height), Label size. Aria-label "Re-pair (open onboarding)".
- Spacer: `flex-1` between Re-pair and ×.
- Dismiss [×]: 32×32 icon button, `<X size={16} />`, `text-[--text-muted]`, `hover:text-[--text]`. Aria-label "Dismiss banner (will return on next 401)".

**Motion:** Enters via 150ms slide-in-from-top + fade. Exits via 100ms fade. Respect `prefers-reduced-motion: reduce` — no slide, no fade; banner appears/disappears instantly.

**Keyboard:**
- Tab order: Re-pair button → × button → next focusable element (theme chip in header is BEFORE the banner because banner mounts below header but tab order follows DOM order; ensure DOM order is `<header>` first, `<RepairBanner>` second, `<main>` third).
- Esc key while focus is anywhere on the banner: triggers `onDismiss`. Does NOT trigger `onRepair`.
- Focus does NOT auto-trap on the banner. User can Tab past it.

**Copy:** See Copywriting Contract row 1.

### `<SchemaDriftState>` (D-08, D-09)

**Purpose:** Inline panel state when a daemon response fails Zod `safeParse` against the corresponding shared schema. Other panels keep working.

**Props:**
```ts
type SchemaDriftStateProps = {
  /** First failing zod issue (rendered prominently) */
  firstIssue: { path: string; expected: string; got: string }
  /** Full zod issue tree (rendered inside <details>) */
  fullIssues: ZodIssue[]
  /** Retries the underlying TanStack Query */
  onReload: () => void
}
```

**Visual (ASCII mockup):**

```
+--------------------------------------------------------------+
|  [!] Schema drift detected                                   |
|                                                              |
|  The agent and dashboard disagree on the shape of this       |
|  response. Update both ends to match.                        |
|                                                              |
|  field:    phaseProgress.evidenceCount                       |
|  expected: number                                            |
|  got:      undefined                                         |
|                                                              |
|  > Show full diff                                            |
|                                                              |
|  [ Reload ]                                                  |
+--------------------------------------------------------------+
```

Tokens & spacing:
- Container: `bg-[--surface]` + `border border-[--border]` + `rounded-md` (6px radius — soft, not chunky). Padding `p-6` (24px).
- Heading row: `flex items-center gap-2`. Icon `<AlertTriangle size={16} />` colored `--danger`. Heading text "Schema drift detected" — Heading size (20px / 600), `text-[--text]`. (Heading is NOT colored danger — color is in the icon only; per "no decorative color" anti-slop rule.)
- Explanation paragraph: Body size, `text-[--text-muted]`. Margin-top `mt-3` (12px).
- Field-path block: `mt-6` (24px) below explanation. Three Code-sized lines, monospace, in a `<dl>`:
  - `<dt>field:</dt><dd>phaseProgress.evidenceCount</dd>` — dt is `text-[--text-muted]` 13px mono; dd is `text-[--text]` 13px mono. Gap between dt and dd: 8px (CSS grid `grid-cols-[max-content_1fr] gap-x-2 gap-y-1`).
  - `<dt>expected:</dt><dd>number</dd>`.
  - `<dt>got:</dt><dd>undefined</dd>`.
- Disclosure: `<details>` with `<summary>Show full diff</summary>`. Summary is Label size, `text-[--accent]` with subtle underline on focus/hover (link-styled). When open, body shows the full Zod issue tree as a `<pre><code>` in 13px mono, `bg-[--surface-elevated]` `p-4` `rounded` block, max-height 240px with `overflow-auto`. `mt-4` from preceding block.
- Reload button: secondary style — `bg-[--surface-elevated] hover:bg-[--border]` + `border border-[--border-strong]`, `text-[--text]`, padding `px-4 py-2` (16/8 → 36px height), Label size. Has lucide `<RefreshCw size={14} />` left-icon with `mr-2` (8px) gap. `mt-6` from preceding.

**Reduced motion:** disclosure open/close uses native `<details>` toggle; no custom animation. Respect inherent `<details>` snap.

**Empty case:** Drift state never renders without a `firstIssue` — the parent decides via `parseOrDrift` helper. If `firstIssue` is undefined the component throws in dev (catches a programming error early).

## Interaction States

Every interactive element exposes the following five states. Tokens reused from the Color section.

| State | Tokens | Notes |
|-------|--------|-------|
| Resting | Element-specific. Default look. | — |
| Hover | `bg-[--surface-elevated]` (or `bg-[--accent-hover]` on accent-filled buttons). Cursor `pointer`. | Pointer device only. Not visible on touch. |
| Focus-visible | `outline: 2px solid var(--ring); outline-offset: 2px;` (or Tailwind `ring-2 ring-[--ring] ring-offset-2 ring-offset-[--bg]`). | Applied via `:focus-visible`, NOT `:focus` — preserves the "no ring on mouse click, ring on Tab" behavior. **Never `outline: none` without a replacement.** |
| Active (pressed) | `brightness-95` filter on hover-tinted bg (slightly darker), or 1px translate-y on buttons (gives a press feel; sub-1ms transition). | Subtle. |
| Disabled | `opacity-50 cursor-not-allowed`. Buttons additionally swap to `bg-[--surface-elevated] text-[--text-subtle]`. | Disabled elements are still in the tab order so screen readers announce them; use `aria-disabled` over `disabled` attribute on form submits to keep them announceable. |
| Loading (CTA only) | Submit button: spinner (lucide `<Loader2 size={14} class="animate-spin" />`) + replace label with in-flight copy. Inputs: `readOnly` (NOT `disabled`). | Cancellation: not supported in Phase 2. (`/health` request is fast; cancel UI would mislead.) |

**Focus ring exceptions:** none. Every focusable element gets the same ring contract. The brand label is a link; it gets the same ring.

**Do NOT use `:hover`-only state to convey information.** Hover reveals only enhance — never replace — the resting state. (Anti-slop: tooltips with critical content fail accessibility.)

## Motion

**Strict budget: 200ms or under, ease-out, single-property only.** Motion is decoration; ban any animation longer than the user's blink.

| Surface | Animation | Duration | Curve | Notes |
|---------|-----------|----------|-------|-------|
| `<RepairBanner>` enter | translateY(-8px) → 0 + opacity 0 → 1 | 150ms | ease-out | Combined transform + opacity is a single perceptual change; allowed. |
| `<RepairBanner>` exit | opacity 1 → 0 | 100ms | ease-in | No translate; a fade-out feels less startling than a slide-out. |
| Theme switch | `<html>` class flip | 0ms (instant) | — | Tailwind v4 `dark:*` utilities snap. **No cross-fade between themes** — that's an AI-slop tell (excessive smoothness). |
| Hover background | `bg-color` 100ms ease-out | 100ms | ease-out | Cheap, GPU-friendly. |
| Focus ring appearance | none (instant) | — | — | Focus rings must appear immediately for accessibility. |
| `<CodeBlock>` copy-success swap | icon swap is instant; the success color holds for 1500ms then reverts via 100ms color transition | 100ms | ease-out | The 1.5s is dwell, not animation. |
| `<details>` disclosure (Show full diff / Why local-only) | native `<details>` toggle (no animation) | — | — | We accept the snap. Smoother require `@starting-style` in v4 — extra implementation surface; defer. |

**Reduced-motion:** every animation listed above is wrapped in `@media (prefers-reduced-motion: no-preference)`. When the user prefers reduced motion: banner enter/exit becomes instant; hover background swap becomes instant; copy-success color revert becomes instant.

**No hero animations. No skeleton shimmer (Phase 2 has no list views). No spinner-on-mount (the SPA is fast enough that a 100ms spinner flicker is worse than no spinner). The only spinner allowed is on the ManualPairForm submit button during the in-flight `/health` call.**

## Accessibility

Beyond the per-component aria notes above, these app-level commitments apply:

| Concern | Implementation |
|---------|----------------|
| Color contrast | All token pairs verified ≥ AA in Color section table. Re-verify if any token shifts during execution. |
| Focus management | `:focus-visible` rings on every interactive element. `outline: none` is BANNED unless a stronger ring replaces it. |
| Keyboard navigation | Tab order = DOM order. `<header>` first, `<RepairBanner>` second when present, `<main>` third. The TanStack Router `<Outlet />` doesn't change tab order. |
| Theme tri-state | `/settings` `<ThemeToggle>` is a native `<input type="radio">` group. Arrow keys cycle. Tab moves into the group on the currently-checked option (W3C ARIA radio behavior). |
| Banner Esc dismiss | Esc keydown listener mounted on the banner; calls `onDismiss`. Listener removed on unmount. Does NOT swallow Esc when banner isn't visible. |
| Form errors | Each `<input>` has `aria-invalid={hasError}` and `aria-describedby={helperOrErrorId}`. The id swaps from helper-text id to error-text id when error appears (single id; content changes). |
| Form-level alerts | `<div role="alert">` on the post-submit error/success region. Re-rendering the region announces the new content. |
| Live-region announces | Copy-success uses `aria-live="polite"` "Copied". Banner appearance announces via the AlertTriangle's containing `<div role="status">` — NOT `role="alert"` (the banner shouldn't yank screen-reader focus mid-task; it's persistent, not transient). |
| Skip link | Standard "Skip to main content" link, visually hidden until focused, jumps to `<main id="main">`. Mounted in `<AppShell>` ahead of `<header>`. |
| Page titles | Per-route `document.title` set by router: "AgenticApps Dashboard — {Route}" (e.g. "— Onboarding", "— Pair", "— Settings", "— Help"). Index uses just "AgenticApps Dashboard". |
| Lang attribute | `<html lang="en">` set in `index.html`. |
| Form labels | Every `<input>` has an associated `<label for="…">`. No placeholder-only labels (placeholder is not a label). |
| Reduced motion | Honored throughout (see Motion). |
| Screen-reader hidden | Decorative arrows ("→") are inside the visible string; `aria-hidden="true"` on lucide icons that duplicate adjacent text labels (e.g. AlertTriangle next to "Schema drift detected" — heading is the label, icon is decoration). |

**Tested with:** keyboard-only navigation, VoiceOver on macOS (default reader), high-contrast mode (forced-colors media query — verify focus rings still visible). Visual regression and live screen-reader testing land in Phase 6 (POLISH-04).

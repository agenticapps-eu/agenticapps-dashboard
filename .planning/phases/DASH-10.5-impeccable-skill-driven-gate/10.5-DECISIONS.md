# Phase 10.5 — Decisions

Decisions adopted as part of Phase 10.5 (impeccable skill-driven gate). Follow the dashboard's existing `D-N-NN` decision-naming convention rather than introducing a standalone ADR directory.

## D-10.5-01 — Retire the CI-enforced impeccable gate

The `.github/workflows/impeccable.yml` workflow, `scripts/check-impeccable-score.mjs`, and its tests are deleted. Phase 6's CI gate has been silently broken since `npx impeccable critique` was removed (v1.0.1, v1.x unpublished from npm). No replacement CI workflow is created.

**Why:**
- The CLI rename is upstream and not reversible — v1.x is gone from npm; only `impeccable detect` exists in v2.x, with a different (finding-list) JSON shape that doesn't produce a 0-100 composite. Adapting the score script to `detect` would be a maintenance burden without measured benefit.
- The CI gate has been broken for weeks with no measured regression — clear evidence it wasn't load-bearing for actual quality.
- The vendored Claude skill `~/.agents/skills/impeccable/SKILL.md` still has `critique` and is strictly more powerful (LLM + persona checks + automated anti-pattern detection in the same pass). Two layers (CI + skill) would have overlapping coverage.

**How to apply:** Delete the three files. Update `CLAUDE.md:38` + `docs/spec/dashboard-prompt.md` lines 554, 594, 696 to describe the per-phase artifact contract. Reference this decision file from each updated line.

## D-10.5-02 — Skill-driven per-phase artifact is the gate

Every frontend-touching phase ends by running `/impeccable critique` against the affected routes at 1440×900 and committing `<N>-IMPECCABLE.md` alongside the phase's other artifacts (`*-REVIEW.md`, `*-QA.md`, `*-VERIFICATION.md`).

The artifact carries:
- Composite score + per-heuristic (Nielsen × 10, scored 0–4 each) + sub-scores
- Cognitive-load checklist results
- Persona red flags (2–3 personas relevant to the surface)
- Priority issues with P0–P3 severity + suggested-command tagging
- AI-slop verdict (both LLM judgment and deterministic detector output)

**Why:**
- The skill is LLM-driven and run-to-run non-deterministic — but the artifact is human-reviewed and committed, so the variability is captured at a single point and judged in context.
- Two-assessment isolation (LLM in one sub-agent, deterministic detector in another, no shared output) is mandated by the skill's `critique.md` reference and preserved by running each in a separate `Agent` invocation.
- Per-phase placement means the artifact lives in commit history next to the code that produced it — better signal than a CI green-check at PR time.

**How to apply:** Add `/impeccable critique` as the canonical Gate 4 step in every phase's HUMAN-UAT / closure checklist. When `/gsd-execute-phase` post-hooks gain skill-invocation support upstream (claude-workflow concern), wire it there.

## D-10.5-03 — Composite floor ≥ 87, calibration pending

Until then, individual phases below 87 should fix-and-rerun before merging, with a written waiver in their VERIFICATION.md if the floor is genuinely unreachable for a phase-specific structural reason (mirroring D-6-09.v1's `/projects/:id` precedent).

## D-10.5-04 — No headless skill invocation in CI

The skill MUST be invoked by a developer in their Claude Code session. No `claude --headless` invocation in CI to automate the per-phase artifact.

**Why:**
- LLM cost per run is non-trivial (sub-agents + browse + source reads).
- Skill is run-to-run non-deterministic; CI flakiness amplifies the cost.
- The artifact is a human-reviewed artefact, not a green-check. Making it CI-enforced removes the review step's value.
- A skipped or absent artifact is itself a visible signal in PR review.

**How to apply:** Phase closure checklist enforcement only. If artifact absence becomes a recurring issue, add a `pre-merge` hook (locally, not CI) that warns when `<N>-IMPECCABLE.md` is missing for a phase that touched `packages/spa/`.

### What the two data points jointly establish

1. **The 87 floor is empirically unreachable for the Coverage matrix surface in its current form**, even after Phase 10's in-phase fixes + Phase 10.6's three-state CTA + Phase 11's polish. Two independent critique runs landed in the same 70s band against the same underlying surface.
2. **The skill-driven gate's natural mid-band sits ~10 points below the CLI-era 87 floor.** D-10.5-03 anticipated this risk explicitly ("there's no a-priori reason it maps cleanly to the skill's Nielsen-heuristic × 10 + AI-slop synthesis"). Confirmed.
3. **The gap is not Phase-quality-driven, it is structural.** The Coverage surface has known categorical limits (matrix-of-status-pills is the canonical "coverage dashboard" gestalt — category-reflex risk: medium) that won't lift past ~80 without reframing the page (e.g., replacing the matrix with a non-tabular representation). The 87 floor implicitly demanded that reframing without naming it.
4. **The Nielsen-24/40 ceiling for this surface in current form is ~30/40** after the 4 inherited P1s land — Assessment A's own estimate. That maps to composite ~82, still below 87.

### Proposed interim recalibration (NOT yet ratified)

Pending the third data point, the candidate recalibrations are:

| Option | Floor | Rationale | Tradeoff |
|---|---|---|---|
| **A — recalibrate to 75** | composite ≥ 75 | Matches empirical mid of observed band (74-76). Phase passes if no new regressions; phases must address fix-and-rerun when below 75. | Loses ambition signal — accepts current surface quality as the new normal. |
| **B — recalibrate to 80, with structural-debt waiver clause** | composite ≥ 80 (or waiver) | Sets the floor above the current 76 to demand the inherited P1s land OR be explicitly waived per-phase. Mirrors D-6-09.v1's `/projects/:id` precedent. | Requires authoring the waiver template; Phase 10.5 already provided this via "written waiver in VERIFICATION.md if the floor is genuinely unreachable for a phase-specific structural reason". |
| **C — keep 87 but redefine "passing"** | composite ≥ 87 OR documented progression | A phase passes if its IMPECCABLE shows lift over the prior phase's score for the same surface (Phase 11's +2 over Phase 10 would qualify) OR a P0 list with named follow-ups. Permanent floor stays aspirational. | Most complex to apply consistently; introduces a new dimension (lift) the gate must check. |
| **D — keep 87, accept that some surfaces structurally cannot reach it** | composite ≥ 87 + explicit per-surface waiver list | The Coverage surface is added to a structural-waiver list (similar to D-6-09.v1's `/projects/:id`). Future surfaces compute against 87 unless on the waiver list. | Mirrors D-6-09.v1's existing precedent most closely; requires authoring the waiver list. |

**Recommended interim direction:** Option B (composite ≥ 80 with per-phase waiver clause). Rationale:

- 80 is above the empirical 76 — preserves the ambition signal that drove the 87 floor in the first place.
- It explicitly invites Phase 11.1 (the recommended inherited-P1 closure phase noted in `11-IMPECCABLE.md`) to demonstrate the lift.
- The waiver clause already exists in D-10.5-03's prose ("a written waiver in VERIFICATION.md if the floor is genuinely unreachable for a phase-specific structural reason") — Option B operationalises it as the normal escape valve rather than the rare exception.
- 80 is the prior-art lower-bound from Phase 06.1's pre-amendment floor (D-6-09.original) before the 87 amendment in D-6-09.v1 — there's a precedent for it.

**Not yet ratified.** A third data point (next frontend phase) will either confirm the band sits in the 75-80 range under Option B (ratify), shift higher after Phase 11.1's structural fixes (consider Option C or D), or surface a third pattern that demands rethinking. See `D-10.5-03.calibration-2` (forthcoming).

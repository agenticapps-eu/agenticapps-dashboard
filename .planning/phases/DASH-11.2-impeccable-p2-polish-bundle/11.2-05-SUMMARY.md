# Phase 11.2 Plan 05: CoverageToolbar Controlled Input Summary

**One-liner:** Converted CoverageToolbar search input from `defaultValue={search}` to a hybrid controlled component using `useState(search)` + `useEffect([search])` mirror-state pattern, fixing URL back-button re-seeding while preserving the 200ms debounce encapsulation.

## Objective

Close P3 #5 from `11.1-IMPECCABLE.md`: the `CoverageToolbar.tsx` search input used `defaultValue={search}`, which only seeds the value on the first render. Subsequent URL changes (browser back-button, deep links, programmatic navigation) left the input out of sync with the URL state.

## Implementation Summary

**Task 1 — RED (38cb585):**

Added a new `describe('controlled input (D-11.2-13)')` block with 4 tests to `CoverageToolbar.test.tsx`:

1. `input renders with value reflecting the search prop` — asserts `input.value === 'dashboard'` after mount with `search="dashboard"`. PASSED (defaultValue seeds on mount).
2. `re-rendering with a different search prop updates the input value` — rerenders with `search="updated"`, asserts `input.value === 'updated'`. **FAILED** (load-bearing test — `defaultValue` only seeds on mount, not on rerender).
3. `typing in the input updates the displayed value immediately (controlled)` — fires change, asserts mirror state updates without waiting for debounce. PASSED.
4. `typing triggers onSearchChange after 200ms debounce` — uses fake timers, asserts debounce fires at 250ms but not at 100ms. PASSED.

**Task 2 — GREEN (3828fad):**

Four minimal edits to `CoverageToolbar.tsx`:

1. Import: `useRef` → `useEffect, useRef, useState`
2. After `debounceRef` declaration: add `const [inputValue, setInputValue] = useState(search)` + `useEffect(() => { setInputValue(search) }, [search])`
3. In `handleSearchChange`: add `setInputValue(value)` call before the debounce timer (synchronous mirror update)
4. On `<input>`: replace `defaultValue={search}` with `value={inputValue}`

Total LOC delta: +12 lines added, 2 lines modified.

## Decisions Made

- **D-11.2-13 implemented:** Hybrid controlled-with-prop-sync pattern chosen over:
  - `key={search}` remount (flickers on URL state-write echo loop)
  - Lifting value to parent (loses local debounce isolation Phase 10 deliberately introduced)
  - The mirror-state pattern preserves debounce locality while making URL back-button changes re-seed the input.

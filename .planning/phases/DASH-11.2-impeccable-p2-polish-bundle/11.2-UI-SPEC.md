## Spacing Scale

Declared values (all multiples of 4 — inherited from Phase 5.1 Tailwind 4 token set):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Tooltip gap-to-trigger (4px), icon gaps |
| sm | 8px | Compact element spacing |
| md | 16px | Toast `px-4`, PageHeader gap |
| lg | 24px | PageHeader `mb-6`, section padding |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

**Exceptions for Phase 11.2:**
- Tooltip trigger-to-panel gap: 4px (top-full + mt-1 = `var(--spacing-xs)`).
- Touch-target minimum: 44×44px (`min-w-[44px] min-h-[44px]`) — Apple HIG mandate.
  Achieved via button `p-[15px]` so that `2 × 15px + 14px icon = 44px` (D-11.2-11).
  The 44px value is NOT a Tailwind spacing token; it is a constraint literal in the
  component only. No 8-point-grid exception needed: 44 ≈ 48px (3×16) which is the
  nearest grid step; the 4px under-run is a hardware-constraint accepted by the
  design contract.
- Tooltip `px-3 py-2` (12px / 8px) — both on scale.

## Component Contracts

### 1. `<Tooltip>` (NEW — `components/ui/Tooltip.tsx`)

**API:**
```tsx
<Tooltip content="Explanation text">{children}</Tooltip>
```

**Props:**
```ts
interface TooltipProps {
  content: string        // Required. Tooltip body copy.
  children: React.ReactNode
}
```

**Trigger rendering:**
- `children` is wrapped in a `<span>` that receives:
  - `tabIndex={0}`
  - `aria-describedby={tooltipId}` (where `tooltipId` is a stable `useId()` value)
  - `className="border-b border-dotted border-text-tertiary cursor-default"`
  - `onMouseEnter`, `onMouseLeave`, `onFocus`, `onBlur` event handlers
  - `onKeyDown` → Escape closes the tooltip

**Floating panel rendering (when open):**
```
position: absolute
top: calc(100% + 4px)   // top-full + mt-1
left: 0
z-index: var(--z-overlay)   // = 100
```
- `id={tooltipId}` (matched by trigger's `aria-describedby`)
- `role="tooltip"`
- className: `absolute top-full mt-1 z-[var(--z-overlay)] bg-card-bg border border-border-subtle rounded-md shadow-card px-3 py-2 text-sm text-text-primary max-w-xs`
- Width: `max-w-xs` (320px) — sufficient for all 4 column-header strings
- No arrow rendering (D-11.2 Claude's Discretion: arrow deferred; column headers
  are mid-table with no overflow risk; absence keeps LOC ~60)

**Open/close timing (D-11.2-03):**
- Open: 100ms delay after `mouseenter` or `focus` (prevents flicker on quick
  cursor pass). Implemented via `setTimeout` ref cleared on `mouseleave`/`blur`.
- Close: 0ms delay on `mouseleave`, `blur`, or Escape key press (snappy hide).

**Animation (D-11.2-04):**
- Opacity-only: `transition-opacity duration-100 ease-out`
- Open: `opacity-0` → `opacity-100` over 100ms
- Close: instant `opacity-0` (0ms, no transition on close)
- Mounted/unmounted via `pointer-events-none` when closed (not DOM-unmounted,
  to avoid re-measuring on re-open)
- Honors `prefers-reduced-motion` implicitly (opacity-only has no motion to suppress)

**NO bounce. NO slide. NO scale.** Phase 11.1 D-11.1-08 motion floor applies.

**Implementation constraints (D-5.1-10):**
- No `cn()`, `clsx`, or CVA
- className string uses inline conditional: `${open ? 'opacity-100' : 'opacity-0 pointer-events-none'}`
- No hex literals
- No Radix UI imports

**Estimated LOC:** ~65 (provider not needed; per-instance state with `useState` + `useRef` + `useId`)

### 6. Controlled search input (`CoverageToolbar.tsx:117-125`)

**Pattern (D-11.2-13 — hybrid controlled with prop sync):**
```tsx
const [inputValue, setInputValue] = useState(search)
useEffect(() => { setInputValue(search) }, [search])

function handleSearchChange(e: React.ChangeEvent<HTMLInputElement>) {
  setInputValue(e.target.value)
  if (debounceRef.current) clearTimeout(debounceRef.current)
  debounceRef.current = setTimeout(() => onSearchChange(e.target.value), 200)
}

// In JSX:
// <input value={inputValue} onChange={handleSearchChange} ... />
// (was: defaultValue={search})
```

The 200ms debounce stays inside `CoverageToolbar` (Phase 10 isolation preserved).
`CoveragePage` receives no API change. URL back-button changes re-seed the input
via the `useEffect([search])` sync.

## States

### Tooltip states

| State | Visual | ARIA | Trigger |
|-------|--------|------|---------|
| Hidden (initial) | panel `opacity-0 pointer-events-none` | `aria-describedby` wired but no announcement | mouse not over trigger, trigger not focused |

| Settled (error) | `<RefreshCw>` static | returns to idle state | — | toast: `Indexing failed: {reason}` (error variant) |

# Phase 11.2 Plan 01: Tooltip Primitive + Column-Header Wiring Summary

**One-liner:** Hand-rolled `<Tooltip>` primitive (opacity-only, tokens-only, ARIA-wired) with `coverageColumnTooltips.ts` SoT closing P1 #1 column-header recognition failure on the Coverage page.

## What Was Built

Three deliverables executed in strict TDD red-green order:

1. **`Tooltip.tsx`** — ~76 LOC hand-rolled React primitive. Opens on mouseenter/focus after 100ms delay (setTimeout ref), closes on mouseleave/blur/Escape instantly. Panel stays mounted when closed (`opacity-0 pointer-events-none`) to avoid remount reflow. ARIA: trigger `aria-describedby={tooltipId}`, panel `role="tooltip" id={tooltipId}`. Z-index `var(--z-overlay)=100`. Zero Radix deps, zero `cn()`/`clsx`/CVA, zero hex literals.

2. **`coverageColumnTooltips.ts`** — 15 LOC SoT file with the 4 D-11.2-05 copy strings for CLAUDE.md, GitNexus, Wiki, and Workflow columns. Lives as a sibling to `coverageColumns.ts`, same `as const` immutability pattern.

3. **`CoverageFamilySection.tsx` wiring** — 4 `<th>` elements (CLAUDE.md, GitNexus, Wiki, Workflow) now wrap their text content with `<Tooltip content={coverageColumnTooltips[key]}>`. Repo and Actions `<th>` remain untouched. Column `<colgroup>` widths unchanged.

## Decisions Made

- **`act()` wrapping for fake-timer assertions** — `vi.advanceTimersByTime()` must be inside `act()` to flush React state updates in JSDOM. The RED test was initially authored without `act()`; GREEN pass required updating the test to use `act(() => { vi.advanceTimersByTime(...) })`. This is the same ergonomic pattern as `Toast.test.tsx`.

- **Comment phrasing to avoid grep false-positive** — The plan's acceptance criteria runs `grep -cE "cn\(|clsx\(|cva\("` on `Tooltip.tsx` and expects 0. The file comment originally said `NO cn()/clsx/CVA` which triggered the grep. Changed to `NO cn / clsx / CVA utility` — same meaning, no false positive.

- **`coverageColumnTooltips.ts` SoT locality** — Placed alongside `coverageColumns.ts` in the `panels/coverage/` directory, not in a shared constants file. Matches D-11.2-05 and the established colocation pattern.

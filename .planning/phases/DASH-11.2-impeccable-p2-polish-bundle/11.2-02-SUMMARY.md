# Phase 11.2 Plan 02: Per-Row In-Flight Feedback + gitnexus-analyze Toast Summary

**One-liner:** Single-mutation fan-out threads `refresh.isPending + refresh.variables` to derive per-row spinner + `aria-busy` on `CoverageRow`, and adds 7th + 8th Toast call sites (gitnexus-analyze success/error) at `CoveragePage`.

## What Was Built

Three components modified in strict TDD red-green order:

1. **`CoverageRow.tsx`** — New optional `pending?: boolean` prop (default `false`). When `true`: `<tr>` gains `aria-busy="true"`, the refresh button gains `disabled`, `aria-busy="true"`, `opacity-100` (forced, no group-hover variants), and the `<RefreshCw>` icon gets `animate-spin`. Normal idle state (`opacity-30 group-hover:opacity-100 focus-within:opacity-100 focus:opacity-100`) is fully preserved when `pending=false`. Touch-target padding (`p-0.5`) unchanged — Plan 04 owns that widening.

2. **`CoverageFamilySection.tsx`** — Two new optional props: `refreshIsPending?: boolean` and `refreshVariables?: CoverageRefreshRequest | undefined`. The `tbody` loop now derives `pending` per-row using the D-11.2-06 formula: `!!(refreshIsPending && refreshVariables?.action === 'gitnexus-analyze' && refreshVariables.family === row.family && refreshVariables.repo === row.repo)`. Only the row whose family+repo matches the in-flight mutation gets `pending=true`; all others get `false`.

3. **`CoveragePage.tsx`** — Two changes:
   - `handleRefresh` `'gitnexus-analyze'` case now wraps `mutateAsync` in try/catch: success fires `toast.show({ message: 'Indexed {family}/{repo}', variant: 'success' })`; error fires `toast.show({ message: 'Indexing failed: {reason}', variant: 'error' })`. `toast` added to `useCallback` dep array.
   - `FAMILIES.map` now threads `refreshIsPending={refresh.isPending}` and `refreshVariables={refresh.variables}` into each `<CoverageFamilySection>`.

## Decisions Made

- **SVGAnimatedString in JSDOM** — SVG `className` is an `SVGAnimatedString` object (not a plain string); `svg?.className ?? ''` returns the object itself (not falsy), making `.toContain('animate-spin')` fail. Fixed by switching all SVG class assertions to `svg?.getAttribute('class')` which returns a plain string. This is a JSDOM ergonomic that the plan's RED tests had not accounted for.

- **`exactOptionalPropertyTypes` and `undefined` literals** — Passing `refreshVariables={undefined}` explicitly (as the test does) fails TypeScript with `exactOptionalPropertyTypes: true` unless the prop type explicitly includes `undefined`. Changed from `refreshVariables?: CoverageRefreshRequest` to `refreshVariables?: CoverageRefreshRequest | undefined`. Same semantics, compliant with strict optional type checking.

- **`getByRole('row')` ambiguity** — A table rendered by `CoverageFamilySection` has both a header `<tr>` and data `<tr>`. `screen.getByRole('row')` throws "Found multiple elements". Fixed by using `screen.getAllByRole('row').find(r => r.querySelector('td'))` to select the data row specifically.

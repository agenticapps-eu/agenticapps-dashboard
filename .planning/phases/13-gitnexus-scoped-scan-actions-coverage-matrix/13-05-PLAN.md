<objective>
**Gap 1 closure (UAT Test 4 — `REPO_NOT_REGISTERED` on the typical row).**

Phase 13 Plan 13-03 wired `<ScanPill>` onto every CoverageRow whose `gitnexus.state ∈ {missing, not-applicable}` and `gitnexusInstalled === true`. But the daemon's `startScan()` resolves the repo's absolute path EXCLUSIVELY via the dashboard project registry (`packages/agent/src/lib/gitnexusScan.ts:150-154`). Coverage discovery is filesystem-driven under `~/Sourcecode/{family}/` (`packages/agent/src/lib/coverageScan.ts:75` via `discoverRepos`), so the render set is a strict superset of the registered set — empirically ~22 rendered rows vs 1 registry entry. Every click on an unregistered row produces an unrecoverable `REPO_NOT_REGISTERED` toast.

**Fix: Option A — add `inRegistry: boolean` to the wire contract, populate it in the scanner, gate the render in the SPA.**

This preserves the T-13-02-01 threat model (Option B was rejected — relaxing the registry-as-allowlist on the daemon would have required a fresh /cso) and does NOT silently auto-register repos (Option C was rejected — hides registration as a side effect). The render gate becomes an honest expression of "scanning this row is structurally possible".

**New architectural decision (surface in 13-DECISIONS or as an addendum to 13-CONTEXT.md):**

> **D-13-EXT-07** — Registry-membership is part of the SPA-bound coverage row contract. The wire schema's `CoverageRow.inRegistry: boolean` flag tells the SPA whether a row's path can be resolved by the daemon's registry-as-allowlist. UI affordances that require daemon-resolvable paths (today: `ScanPill`; future: any per-row mutation that calls `startScan`/equivalent) MUST gate on it. This codifies what UAT Test 4 surfaced: the filesystem-discovery set is a strict superset of the registry-membership set, and any daemon-write affordance attached to a Coverage row must honour the membership precondition.

Four TDD-driven tasks:
1. Extend the shared `CoverageRowSchema` with `inRegistry: boolean` and unit-test the schema. **Update existing valid-row fixtures FIRST, then add the new omission/type test against a local fixture.**

3. Update `CoverageRow.tsx` render gate to require `row.inRegistry === true`; add the matching component-level gate test in `CoverageRow.test.tsx`.
4. Audit & repair all SPA + agent test fixtures that construct `CoverageRow` objects so the new required field is set; gate green via `pnpm -r test --run`. ALSO: append D-13-EXT-07 to 13-CONTEXT.md as part of this task and gate via grep.

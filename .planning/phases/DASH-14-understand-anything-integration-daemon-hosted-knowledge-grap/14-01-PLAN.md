<objective>
Extend the shared wire-schema package with the three Phase 14 contracts: the `understand` block on `HealthResponseSchema` (D-14-02 version-drift surface), the `understand` optional column on `CoverageRowSchema` (D-14-08 staleness + per-row scoped viewer token carrier per D-14-03), and `buildUnderstandCommand()` (D-14-10 copy-pill command builder).

Purpose: every downstream plan (daemon scanner, daemon route, SPA column, SPA page) validates against these shapes. Shipping them first unblocks three parallel Wave-2 plans.
Output: extended shared schemas + tests, barrel exports updated.

PLANNER DECISION (Claude's discretion, documented per CONTEXT.md): the scoped viewer token is **per-repo**, carried in the coverage row's `understand.viewerToken` field — NOT a single global token in `/health`. Rationale: the upstream viewer fetches `/knowledge-graph.json` root-absolute with only `?token=`; the token is the ONLY carrier of repo identity at the data endpoints, so it must be repo-bound (HMAC over repoId, minted in plan 14-02). 14-RESEARCH.md Critical Q1 vs Q4 contradict on this point; Q1's per-repo conclusion is the correct one. `/health.understand` therefore carries NO token (also avoids RESEARCH Pitfall 6 token-in-health logging leak).
</objective>

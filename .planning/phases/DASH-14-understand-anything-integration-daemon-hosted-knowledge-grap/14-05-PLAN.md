<objective>
Re-implement the upstream understand-anything dev-server contract in Hono: serve the installed static viewer at `/understand/{family}/{repo}/` and the 6 data endpoints (`knowledge-graph.json`, `meta.json`, `config.json`, `domain-graph.json`, `diff-overlay.json`, `file-content.json`) at the app ROOT, token-gated by the per-repo scoped viewer token.

Purpose: the click-to-viewer experience (D-14-07) — the centerpiece of Phase 14.
Output: understandViewer.ts route module + app.ts mounting + the phase's core security test suite.

ARCHITECTURE (binding — from 14-RESEARCH.md Critical Q1, verified against upstream source):
- The viewer's data fetches are ROOT-ABSOLUTE (`/${fileName}?token=…` hardcoded in App.tsx/TokenGate.tsx/CodeViewer.tsx). The 6 data endpoints therefore MUST be registered at the top level of the Hono app, NOT under `/understand/...`. The `?token=` (verified by `verifyViewerToken`, plan 14-02) is the ONLY carrier of repo identity at those endpoints.
- Static assets are served under `/understand/{family}/{repo}/` (repoId = two path segments). The installed bundle is built with `--base=./` (plan 14-07), so `index.html` must be served at a TRAILING-SLASH URL — requests to `/understand/{family}/{repo}` (no slash) MUST 301/302-redirect to the trailing-slash form, otherwise `./assets/…` chunk URLs resolve one segment short and 404.
- Static assets (index.html + chunks) require NO token: the browser's chunk requests carry no query string, and the bundle contains zero project data (upstream's TokenGate is client-side and validates by fetching `/knowledge-graph.json?token=`). All PROJECT DATA flows through the 6 token-gated endpoints. This narrows the 14-PATTERNS.md sketch (which gated `/*`) — gating assets would break chunk loading.
- Both routers mount AFTER the CORS middleware and BEFORE `app.use(bearerAuth(...))` in createApp() (Hono registration-order short-circuit, RESEARCH Pitfall 3).
- D-14-04: no bindMode check anywhere in this module — full Tailscale parity (deliberate contrast with D-13-11; read-only surface).
- INV-04 note: the data endpoints pass through UPSTREAM artifacts (graph/meta/config JSON) after sanitization; they are not wrapped in outbound() Zod parsing because the upstream graph schema is not ours to pin — drift there must not 500 the viewer. Error responses use the upstream's exact `{error: string}` shapes.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → /understand/* static | Unauthenticated static bundle serving (no project data) |
| browser ?token= → 6 data endpoints | Untrusted token + untrusted ?path= cross here; project file contents flow back |
| graph JSON → browser | Upstream-generated artifact crosses to the wire; may embed absolute paths |
| repoId → filesystem | Decoded repoId selects the repo root being read |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-14-05-01 | Spoofing (token gate bypass) | 6 data endpoints | mitigate | verifyViewerToken on every data route; bearer token explicitly NOT accepted (Task 1 Test 1); mount-order integration test (Task 3 Test 5) |
| T-14-05-02 | Tampering (path traversal in file-content) | /file-content.json | mitigate | 12-guard upstream suite in order + graph-membership allow-list + 1 MiB cap + binary rejection (Task 2, D-14-05) |
| T-14-05-03 | Elevation (symlink escape) | file-content resolution | mitigate | realpath containment under realpath(repo root) even for graph-listed entries (Task 2 Test 4) |
| T-14-05-04 | Information Disclosure (home-layout leak) | knowledge-graph + domain-graph responses | mitigate | FIX 2 sanitisation both endpoints + negative assertions on root/homedir substrings (Task 1 Tests 3/5, D-14-05b) |
| T-14-05-05 | Tampering (viewer-asset path traversal) | static serving | mitigate | Segment regex pre-validation + serveStatic traversal handling (or explicit realpath containment in the fallback impl) + encoded-traversal tests (Task 3 Tests 3/6) |

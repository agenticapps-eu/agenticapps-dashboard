Purpose: one explicit, user-driven build step; the daemon only ever serves prebuilt assets (mirrors install-launchd's CLI-side-effect pattern).
Output: CLI module + tests + commander wiring.

Build sequence (from 14-RESEARCH.md Critical Q2, verified against the live cache):
1. Locate `~/.claude/plugins/cache/understand-anything/understand-anything/` → newest semver version dir (via viewerInstall helpers).
2. Check `pnpm` on PATH.
3. Ensure `packages/core/dist/schema.js` exists; else `pnpm build` in packages/core (after `pnpm install` if node_modules absent — RESEARCH assumption A5 mitigation).

5. Copy dist/ recursively to `~/.agenticapps/dashboard/understand-viewer/<version>/` (write boundary).
6. Print success + restart hint.
</objective>

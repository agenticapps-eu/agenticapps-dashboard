<objective>
Lay the agent-side foundation for the understand viewer: extract `deterministicRepoRoot()` to a shared module (D-14-09 — do NOT duplicate the realpath logic), build the scoped viewer-token system (D-14-03), and build viewer-install detection helpers (D-14-01/D-14-02) consumed by the health route, the viewer route, and the CLI installer in wave 3.

Purpose: three wave-3 plans (14-05 route, 14-06 scanner/health, 14-07 CLI) all import from these modules.
Output: three new lib modules + tests, constants additions, boot + rotation wiring.

PLANNER DECISION — viewer token design (resolves the 14-RESEARCH.md Q1-vs-Q4 contradiction; see 14-01 objective): tokens are PER-REPO and self-describing. Format: `v1.<base64url(repoId)>.<hex hmac-sha256(secret, repoId)>`. The secret is 32 random bytes stored hex-encoded in `~/.agenticapps/dashboard/viewer-token.json` (0600, atomic write). `verifyViewerToken()` recovers the repoId from the token itself — required because the upstream viewer's data fetches are root-absolute (`/knowledge-graph.json?token=…`) and carry no repoId. The repoId inside the token is not secret (it already appears in the viewer URL path); the HMAC binds it so a token for repo A cannot read repo B. Deterministic minting means tokens survive daemon restarts and need no per-repo storage.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser ?token= → daemon | Untrusted token string is the sole carrier of repo identity for viewer data endpoints |
| repoId (decoded from token) → filesystem | Decoded repoId selects which repo root the daemon reads |
| ~/.agenticapps/dashboard/viewer-token.json | Secret at rest; compromise forges any repo's viewer token |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-14-02-01 | Spoofing | verifyViewerToken | mitigate | HMAC-SHA256 over repoId with 256-bit secret; length-guard + timingSafeEqual compare; uniform null on failure (Task 2 tests 4/5/7) |
| T-14-02-02 | Tampering (repoId injection) | verifyViewerToken decoded repoId | mitigate | D-13-EXT-11 two-layer regex+refine applied to the DECODED repoId before any FS use (Task 2 test 4) |
| T-14-02-03 | Elevation (symlink escape) | resolveRepoRoot / deterministicRepoRoot | mitigate | Extraction preserves realpathSync.native guard + family allow-list verbatim; migrated security suite must stay green (Task 1) |
| T-14-02-04 | Information Disclosure | viewer-token.json secret at rest | mitigate | 0600 atomic write + assertSecurePermissions refusal on loose modes (Task 2 tests 1/2), inside the daemon-write boundary |
| T-14-02-05 | Spoofing (stale tokens) | rotation | mitigate | rotateViewerSecret hooked into rotateToken — bearer rotation invalidates viewer tokens (Task 2 test 6) |
| T-14-02-SC | Tampering | npm installs | accept | No new packages (node:crypto only) |
</threat_model>

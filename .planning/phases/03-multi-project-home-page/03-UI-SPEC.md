# Phase 3 — UI Design Contract

> Visual and interaction contract for the multi-project home page. Inherits Phase 2's full design baseline unchanged; this document specifies only the Phase 3-new surfaces. Generated by gsd-ui-researcher, verified by gsd-ui-checker. Anti-AI-slop discipline (D-42/D-43/D-44) is non-negotiable — the impeccable:critique ≥ 90 gate in Phase 6 measures what Phase 3 ships.

| Property | Value |

## Spacing Scale

_Source: Phase 2 UI-SPEC (inherited unchanged). Phase 3 activates the `3xl` token previously reserved._

| Token | Value | Tailwind class | Usage |
|-------|-------|----------------|-------|
| 0 | 0px | `p-0` / `gap-0` | Reset only |
| xs | 4px | `p-1` / `gap-1` | Icon ↔ text gap inside button or chip; tight inline chip padding; kebab button icon gap |
| sm | 8px | `p-2` / `gap-2` | Card internal gap between data rows; chip-to-chip gap in toolbar; tag chip gap |
| md | 16px | `p-4` / `gap-4` | Card default padding (horizontal); modal panel body padding; context menu item padding (vertical only); grid column gap |
| lg | 24px | `p-6` / `gap-6` | Card padding (generous variant for modal chrome); toolbar section gap; modal step padding |
| xl | 32px | `p-8` / `gap-8` | Grid row gap; page main padding-top on home route |
| 2xl | 48px | `p-12` / `gap-12` | Page-level top padding on home page below header |
| 3xl | 64px | `p-16` / `gap-16` | Reserved; not used in Phase 3 active layouts |

**Home page grid container:** `max-w-5xl` (1024px) — wider than Phase 2's `max-w-3xl` because the card grid needs three columns on a 1280px viewport. Page horizontal padding: `px-6` (24px) on mobile, `px-8` (32px) ≥ md breakpoint. The `AppShell` `<main>` currently uses `max-w-3xl`; Phase 3 must override this for the home route only (either a per-route className passed down from the route component, or a dedicated `HomeLayout` wrapper that sets `max-w-5xl` while other routes keep `max-w-3xl`).

**Card grid:** `grid grid-cols-1 sm:grid-cols-2 lg:grid-cols-3 gap-4`. Each cell has fixed layout; the card fills its cell. No masonry — use `grid-rows` CSS to give each card a `min-height: auto` (natural height) while the grid row height adapts to the tallest card in that row. Cards do NOT use `overflow: hidden` on the outer grid cell so the hover-expand can reflow naturally within the card's flex column.

**Touch/hit-target exception:** Same as Phase 2 — all interactive elements ≥ 32×32px. Kebab button: 32×32px (`h-8 w-8 p-2` with 16px icon). Filter chips: `px-3 py-1.5` is NOT permitted; use `px-3 py-2` (12/8 → 36px height). Context menu items: `px-4 py-2` minimum (16/8 → 36px touch target). **No `*.5` Tailwind utilities** (no `py-1.5`, `px-2.5`, etc.).

## Typography

_Source: Phase 2 UI-SPEC (inherited unchanged — four sizes, two weights)._

| Role | Size | Weight | Line Height | Family | Tailwind | Phase 3 Usage |
|------|------|--------|-------------|--------|----------|---------------|
| Label | 14px | 400 / 600 | 1.4 (≈20px) | system-sans OR system-mono | `text-sm` + weight/family variant | Card name subtitle (client · tag), card footer timestamps, toolbar chip labels, sort dropdown label, context menu items, modal field labels and helper text, command palette action labels, filter chip counts `(N)`, finding glyph aria-labels |
| Body | 16px | 400 | 1.5 (24px) | system-sans | `text-base font-normal leading-relaxed` | Modal step explanation text, "no markers detected" hint, blocked-path explanation |
| Heading | 20px | 600 | 1.3 (26px) | system-sans | `text-xl font-semibold leading-snug` | Card project name (compact default), modal step headings ("Register a project", "Confirm registration"), command palette dialog heading |
| Display | 28px | 600 | 1.2 (≈34px) | system-sans | `text-3xl font-semibold leading-tight tracking-tight` | Not used in Phase 3. Inherited from Phase 2 (OnboardingHero only). |

**Phase 3 typography rules (additive to Phase 2):**
- Card project name renders at Heading size (20px / 600) in compact state. This is the single largest text element per card — keeps scannability.
- Card data rows (phase status, finding counts, TDD pairs, verification, last commit) all render at Label size (14px). Finding count glyphs render at Label size with `font-mono` so numeric columns align across cards.
- Finding-count numbers render as: `font-mono text-sm text-[--text]` for the count digit; the Unicode glyph is inline before the number with 4px gap (`gap-1`). No color applied to the glyph itself (Unicode emoji carry their own color rendering; `text-[--text]` applies to the surrounding text only).
- Command palette input: Body size (16px) for the search input text; Label size (14px) for action-list rows.
- No 5th font size. No 12px. No `text-xs` in Phase 3 components.

### `<HomeToolbar>`

**Purpose:** Filter chips + search box + sort dropdown. (D-38, D-39, D-40)

**Layout (ASCII):**
```
[all] [active] [client] [internal] [wip(2)] ...   [Search...          ] [Sort: Recommended ▾]
```

Flex row, `flex items-center gap-3 flex-wrap`. Filter chips on the left, search + sort anchored right via `ml-auto` on the search group.

**Filter chips:**
- Fixed set (always present, in this order): `all`, `active`, `client`, `internal`.
- Overflow chips: any tag in the registry union that is not one of those four, appended after the fixed set. Each overflow chip shows a `(N)` count suffix in `text-[--text-muted]` Label size: e.g. `wip (2)`.
- **Unselected chip:** `bg-[--surface-elevated] text-[--text] border border-[--border]` + `px-3 py-2 rounded-md text-sm font-semibold`. Hover: `bg-[--border]`.
- **Selected chip:** `bg-[--accent] text-[--accent-fg] border border-[--accent]`. No hover color change when selected (the fill already communicates selection).
- **`all` chip behavior:** When `all` is active, it is the sole selected chip. Clicking any other chip deselects `all` and selects the new chip. Clicking `all` deselects any active chips and returns to the default show-all state.
- **Multi-select:** Multiple non-`all` chips can be active simultaneously. Semantics: OR (a project matching any selected chip is shown).
- Focus: standard `ring-2 ring-[--ring] ring-offset-2 ring-offset-[--bg]` on `:focus-visible`.

**Search box:**
- `<input type="search" placeholder="Search projects…" aria-label="Search projects">`.
- Width: `w-48 md:w-64` (192px / 256px). Expands to `w-full` on small viewports.
- Styling: `bg-[--surface-elevated] border border-[--border-strong] rounded-md px-3 py-2 text-sm text-[--text] placeholder:text-[--text-subtle]`.
- Icon: `<Search size={14} />` rendered as a left-side decorative icon inside the input wrapper (`absolute left-3`, `pl-8` on the input). Color `text-[--text-muted]`.
- Behavior: Lower-cased substring match on `name + client + tags + currentPhase dirname`. AND with active chip filter. `Esc` clears the value and returns focus to the input.
- No debounce — filtering is client-side and synchronous on the cached registry list.

**Sort dropdown:**
- Native `<select>` element. Consistent with Phase 2's preference for native HTML over custom listboxes wherever the UX cost is acceptable. A sort dropdown opens infrequently and native `<select>` is accessible everywhere.
- Label: visually hidden `<label for="sort-select">Sort by</label>` (screen readers get the context; sighted users see the current value in the select).
- Wrapper: `relative inline-flex items-center gap-1`.
- Select styling: `bg-[--surface-elevated] border border-[--border-strong] rounded-md pl-3 pr-8 py-2 text-sm text-[--text] appearance-none cursor-pointer`. Chevron: absolute-positioned `<ChevronDown size={12} />` in `text-[--text-muted]` inside the wrapper, `pointer-events-none`.
- Options (in order): `Recommended` (value `recommended`, default), `Last commit ↓` (value `lastCommit`), `Name ↑` (value `name`), `Phase ↓` (value `phase`), `Client ↑` (value `client`).
- Selection is session-only — no localStorage persistence.

### `<ProjectCard>`

**Purpose:** Single registered project card. Compact default; hover/focus expands inline. (D-34, D-35, D-36, D-37, D-43, D-44)

**Card anatomy (compact — always visible):**
```
┌─────────────────────────────────────────────┐
│ Project Name                            [⋮] │  ← Heading size (20px/600), kebab always-on
│ Client Name · client                        │  ← Label (14px/400), text-muted
│                                             │
│ Phase 04 · In Progress                      │  ← Label (14px/600)
│                                             │
│ Stage 2:  🔴 0  🟡 2  🟢 5                  │  ← Label (14px), finding row
│                                             │
│ last commit 14m ago                         │  ← Label (14px/400), text-muted, footer
└─────────────────────────────────────────────┘
```

**Card anatomy (expanded — visible on hover/focus, inline addition):**
```
│ Stage 1:  🔴 0  🟡 0  🟢 1                  │  ← Only when stage1.ran
│ DB-AUDIT: 0 critical · 1 high · 0 med       │  ← Only when dbAudit non-null
│ TDD pairs: 4/5                              │  ← Only when tdd non-null
│ Verification: 3/4                           │  ← Only when verification non-null
│ Branch: feat/impeccable                     │  ← Only when branch non-null
```

**Card chrome:**
- Container: `<button type="button">` (click = navigate). `bg-[--surface] border border-[--border] rounded-md p-4 text-left w-full relative flex flex-col gap-2`.
- Hover: `hover:bg-[--surface-elevated] hover:border-[--border-strong]`.
- Focus-visible: standard ring.
- Transition: `transition-colors duration-100 ease-out` (100ms, matches Phase 2 hover bg contract).

**Compact height:** Natural content height (~5 lines at Label size with `gap-2` = approximately 120–140px). The card does NOT have a fixed `min-height` — it sizes to content.

**Hover-expand transition:**
- The expanded section wraps all additional rows in `<div class="overflow-hidden">`.
- Default (not expanded): `max-height: 0; opacity: 0`.
- Expanded (hover/focus): `max-height: 160px; opacity: 1` (160px is generous upper bound for 5 extra Label rows at 20px each with gaps).
- CSS: `transition: max-height 120ms ease-out, opacity 100ms ease-out`. No `transform`, no `scale`, no `rotate`, no bounce (`ease-out` not `spring`).
- Expanded state triggers on `group-hover` and `group-focus-within` using Tailwind's `group` pattern on the card container.
- `prefers-reduced-motion: reduce`: `max-height` and `opacity` snap instantly (no transition). Use `motion-safe:transition-[max-height,opacity]` to scope the transition.

**Finding count rendering (D-36):**
- Row label: Label size, `text-[--text-muted] font-semibold`. e.g. "Stage 2:".
- Glyph groups: `🔴 0  🟡 2  🟢 5` — each glyph is a `<span>` with `aria-hidden="true"`. The containing group has a wrapping `<span aria-label="0 critical, 2 medium, 5 low">` for screen readers.
- Glyph font-size: `text-sm` (14px Label tier) — glyphs render at their natural emoji size relative to this font size.
- Numbers: `font-mono text-sm text-[--text]` — monospace so numeric columns align across cards.
- Spacing: `gap-2` (8px) between the three glyph+number pairs. Fixed-width layout: wrap each pair in `<span class="inline-flex items-center gap-1 w-12">` so all cards align at the same column positions regardless of digit count (e.g. `0` and `12` both take the same width).
- DB-AUDIT row (expanded only): textual labels, not glyphs. `0 critical · 1 high · 0 medium · 0 low`. All Label size, `text-[--text]`. Separator `·` in `text-[--text-muted]`.

**Phase status rendering:**
- "Phase 04 · In Progress" — phase number comes from the phase directory name (e.g. `04-single-project-view`); extract the leading digits. Status badge: rendered as inline text with a dot separator, NOT a pill badge. Color: status text uses `text-[--text]` at Label weight 600. No color-coded phase status — information is in the text, not in a color fill.
- Empty-phase card (D-44): Replace the phase line with `no .planning/` in `text-[--text-muted]` and an inline `<a>install workflow skill →</a>` link in `text-[--accent]` with `text-sm` underline. No finding rows. Click-through still works.

**Loading state while `/api/projects/{id}/overview` is in-flight:**
- Card renders the skeleton rows as static `—` dashes in `text-[--text-muted]`. No shimmer, no animated placeholder. All compact rows present but showing `—`. Expanded section hidden (not triggered by hover while loading).

**Error states:**
- 5xx / network failure: compact rows show `—`. Footer changes to `⚠ overview unavailable · retrying` — `<AlertTriangle size={12} />` in `text-[--warning]` + Label text in `text-[--text-muted]`. Card still navigates on click.
- Schema drift: compact rows replaced by `<SchemaDriftState />` inline (Phase 2 component, reused). Card height expands to accommodate it. Click-through still works.
- Unreachable project (D-06): card renders with `opacity-60`. Phase line replaced by `unreachable: <root>` — `<span class="text-[--danger] text-sm font-mono">unreachable: {root}</span>`. Inline "Unregister?" link in `text-[--danger] text-sm` that opens the context menu's unregister flow. Card sorted last within its tag bucket regardless of sort key. Finding rows absent. Click-through still works (the placeholder route).

**Kebab button (always-visible, D-23):**
- `<button type="button" aria-label="Project options for {name}" class="absolute top-3 right-3 ...">`.
- Size: `h-8 w-8 p-2` with `<MoreVertical size={16} />`.
- Styling: `text-[--text-muted] rounded-md hover:bg-[--surface-elevated] hover:text-[--text]`.
- Focus: standard ring.
- Click: stops propagation (prevents card navigate) and opens `<CardContextMenu>` anchored to the button.
- Z-index: `z-10` relative to the card to sit above the card's hover state.

**Right-click / long-press (D-23):**
- `onContextMenu` on the card's button container: stop default, open `<CardContextMenu>` at pointer position.
- Long-press (Pointer Events): `pointerdown` starts a 500ms timer; `pointerup` / `pointermove` (> 8px threshold) cancels it. On timer fire: open context menu. No visual feedback during press accumulation (consistent with D-43 no-animation rule).

### `<CardContextMenu>`

**Purpose:** Accessible context menu shared by right-click / long-press / kebab. (D-23)

**Trigger mechanism:** Rendered as a portal (mounted at `document.body` to escape card stacking context). Positioned absolutely at the pointer coordinates or kebab button coordinates.

**Chrome:**
- Container: `bg-[--surface] border border-[--border-strong] rounded-md shadow-md py-1 min-w-[160px] z-50`.
- No header. No title. Menu role: `role="menu"`.

**Items:**
1. `Rename` — `<button role="menuitem">`. Icon: `<Tag size={14} />`. Label: "Rename".
2. `Edit tags` — `<button role="menuitem">`. Icon: `<Tag size={14} />`. Label: "Edit tags".
3. `Unregister` — `<button role="menuitem">`. Icon: `<Trash2 size={14} />`. Label: "Unregister". Color: `text-[--danger]` for both icon and label. Confirms inline (see below).

**Item styling:**
- Resting: `flex items-center gap-2 px-4 py-2 text-sm text-[--text] w-full text-left hover:bg-[--surface-elevated]`.
- Unregister item: `text-[--danger] hover:bg-[--danger-surface]`.
- Focus: `focus-visible:bg-[--surface-elevated] focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-inset focus-visible:ring-[--ring]`.

**Keyboard navigation:**
- `ArrowDown` / `ArrowUp` move focus between items (roving tabindex pattern).
- `Enter` / `Space` activate the focused item.
- `Esc` closes the menu and returns focus to the trigger (kebab button or card).
- `Tab` closes the menu (focus leaves the menu).
- First item receives focus on open.

**Unregister inline confirmation:**
- Clicking "Unregister" does NOT close the menu and open a second dialog. Instead, the menu replaces its items with an inline confirmation:
  - Heading: "Unregister {name}?" — Label size, `text-[--text] font-semibold`. `px-4 py-2`.
  - Subtext: "This only removes it from the dashboard. No files are deleted." — Label size, `text-[--text-muted]`. `px-4 pb-2`.
  - Two buttons side by side: `[Cancel]` (secondary style) + `[Unregister]` (danger style: `bg-[--danger] text-white hover:bg-red-700`). `px-4 py-2 gap-2`.
  - Cancel returns to the normal item list. Confirm fires the `useUnregister(id)` mutation and closes the menu.

**Dismiss:** Click outside, `Esc`, or focus leaving the menu container (via `blur` + `relatedTarget` check) closes it.

### `<RegisterModal>`

**Purpose:** Two-step in-place registration modal. Native `<dialog>` element. (D-20, D-26, D-27, D-28, D-29, D-30, D-31)

**Step 1 — path input:**
```
┌─────────────────────────────────────────────────────┐
│  Register a project                            [×]  │
│                                                     │
│  Project path                                       │

│  │ /Users/donald/Sourcecode/my-project         │   │
│  └─────────────────────────────────────────────┘   │
│  Full path to the project root.                     │
│                                                     │
│  [ Preview ]                                        │
└─────────────────────────────────────────────────────┘
```

**Step 2 — confirm:**
```
┌─────────────────────────────────────────────────────┐
│  Register a project                            [×]  │
│                                                     │
│  Resolved path                                      │
│  /Users/donald/Sourcecode/my-project                │
│                                                     │
│  Name                                               │
│  ┌─────────────────────────────────────────────┐   │
│  │ my-project                                  │   │
│  └─────────────────────────────────────────────┘   │
│                                                     │
│  Client (optional)                                  │
│  ┌─────────────────────────────────────────────┐   │
│  │                                             │   │
│  └─────────────────────────────────────────────┘   │
│                                                     │
│  Tags                                               │
│  [active] [client] [+ add tag]                     │
│                                                     │
│  ⓘ No git repo or .planning/.claude found here.    │
│     Cards may show empty data.                      │
│                                                     │
│  [ Back ]   [ Confirm ]                             │
└─────────────────────────────────────────────────────┘
```

**Chrome:**
- Native `<dialog>` element, opened via `.showModal()`.
- Max-width: `max-w-md` (448px). Full-width on mobile (`w-full mx-4`).
- Dialog chrome: `bg-[--surface] border border-[--border-strong] rounded-lg p-6`. No drop shadow in dark mode (border provides separation); subtle `shadow-lg` in light mode only (use `dark:shadow-none`).
- Backdrop overlay: `::backdrop` pseudo-element styled with `background: rgba(0,0,0,0.6)`. Both themes use this value (the page content behind is already dark in dark mode; 60% opacity is sufficient in both themes).
- Focus trap: manual (no external library). `Tab` and `Shift+Tab` cycle through the modal's focusable descendants only.

### `<CommandPalette>`

**Purpose:** Global Cmd/Ctrl+K action palette. Mounted in `<AppShell>`. (D-32)

**Trigger:** `keydown` listener on `window` matching `(e.metaKey || e.ctrlKey) && e.key === 'k'`. Listener mounted once in `AppShell`, cleaned up on unmount.

**Chrome:**
- Native `<dialog>` element, opened via `.showModal()`.
- Max-width: `max-w-lg` (512px). Centered vertically at 20% from top (not perfectly centered — user's eyes are in the upper-center of the screen). Use `::backdrop` same as `<RegisterModal>`.
- Dialog styling: `bg-[--surface] border border-[--border-strong] rounded-lg overflow-hidden`. No `p-*` on the dialog itself — padding comes from child sections.
- Internal layout: `flex flex-col`.

**Palette anatomy:**
```
┌────────────────────────────────────────────────────┐
│  [🔍] Search or jump to…                     [⌘K] │  ← input row
├────────────────────────────────────────────────────┤
│  Register project                                  │  ← action row
│  Jump to acme-app                                  │  ← action row (per registered project)
│  Jump to beta-app                                  │
│  Refresh data                                      │
│  Toggle theme                                      │
└────────────────────────────────────────────────────┘
```

**Input row:**
- `<input type="search" placeholder="Search or jump to…" aria-label="Command palette search" aria-owns="palette-listbox">`.
- Styling: `w-full bg-transparent border-0 px-4 py-3 text-base text-[--text] placeholder:text-[--text-subtle] focus:outline-none`.
- `<Search size={16} />` icon left of input: `absolute left-4 text-[--text-muted]` — same pattern as HomeToolbar search.
- `⌘K` hint chip: `absolute right-4 top-1/2 -translate-y-1/2`. Styling: `bg-[--surface-elevated] border border-[--border] rounded px-1.5 py-0.5 font-mono text-xs text-[--text-muted]`. **Exception to the no-`*.5` rule:** `px-1.5 py-0.5` is permitted for the ⌘K badge only — it is a decorative hint chip, not a spacing token. This exception is annotated inline.
- Separator below input row: `border-b border-[--border]`.

**Listbox:**
- `<ul id="palette-listbox" role="listbox" aria-label="Actions">`.
- `max-h-72` (288px) with `overflow-y-auto`.

**Action rows:**
- Each: `<li role="option" id="palette-option-{i}" aria-selected={focused}>`.
- Styling: `flex items-center gap-3 px-4 py-3 text-sm text-[--text] cursor-pointer`.
- Focused (keyboard): `bg-[--surface-elevated]`.
- Hover (mouse): `bg-[--surface-elevated]`.
- No leading icon on action rows (consistent with D-43 minimalism — icons add visual noise in a dense list). Exception: "Jump to {name}" rows may optionally prepend `<FolderOpen size={14} class="text-[--text-muted]" />` to distinguish project-jump actions from global actions. Planner decides — either zero icons or all-jump-rows-only icons; mixed is prohibited.

**Keyboard navigation:**
- `ArrowDown` / `ArrowUp`: move focus through rows. Updates `aria-activedescendant` on the input.
- `Enter`: activate focused row.
- `Esc`: close palette, return focus to previously focused element.
- `Tab`: close palette (focus leaves).
- When input has no filter text and the palette opens, first row is highlighted by default.

**Fuzzy filter:** Lower-cased substring match on action label + project names. No external library. Typing in the input narrows the list in real time. When filter produces zero matches: render a single non-interactive row "No actions found." — Label size, `text-[--text-muted]`, `px-4 py-3`.

**Focus management:** On open, focus moves to the search input. On close, focus returns to the element that was focused before the palette opened (capture `document.activeElement` on `keydown` before `.showModal()`).

**Actions (v1):**
1. "Register project" — opens `<RegisterModal>`, closes palette.
2. "Jump to {name}" — one row per project in the registry. Navigates to `/projects/{id}`, closes palette.
3. "Refresh data" — calls `queryClient.invalidateQueries({ queryKey: ['registry'] })` + invalidates all `['overview', *]` queries, closes palette.
4. "Toggle theme" — calls the `useTheme().setChoice(nextTheme)` cycle, closes palette.

## Accessibility

Phase 2 commitments apply unchanged. Phase 3-specific additions:

| Concern | Implementation |
|---------|----------------|
| Card click target | `<button type="button">` spanning the full card. Aria-label: "View {name}". Right-click and long-press stop propagation before reaching the button's click handler. |
| Kebab button | `aria-label="Project options for {name}"`. Always-visible, always in tab order. `aria-haspopup="menu"`. |
| Context menu | `role="menu"`, each item `role="menuitem"`. Focus moves to first item on open. `Esc` returns focus to kebab or card root. Roving tabindex for arrow-key navigation. |
| Finding glyphs | Unicode emoji (`U+1F534`, `U+1F7E1`, `U+1F7E2`) are `aria-hidden="true"`. Containing span carries `aria-label="N critical, N medium, N low"`. |
| Register modal | Native `<dialog>` provides `aria-modal="true"` semantics automatically. Focus trap is manual. Heading `<h2>Register a project</h2>` gives the dialog its accessible name. |
| Modal close | After close, focus returns to the element that opened the modal (register card button or kebab menu trigger). |
| Command palette | `<dialog>` + `<input aria-owns="palette-listbox">` + `<ul role="listbox">` + `aria-activedescendant` on input tracks focused row. |
| Card loading state | `aria-busy="true"` on the card while `/overview` is loading. Screen readers will not announce the `—` placeholder values as meaningful data. |
| Card error | `role="status"` on the footer `⚠ overview unavailable` text, scoped to the card. |
| Unreachable card | The "Unregister?" link has `aria-label="Unregister {name}"` to provide context without repeating surrounding text. |
| Filter chip group | Wrap chips in `<div role="group" aria-label="Filter by">`. Each chip is `<button type="button" aria-pressed={selected}>`. |
| Search input | `<input type="search">` with `aria-label="Search projects"`. `Esc` clears value. |
| Sort select | Visible `<label>` (visually hidden with `sr-only`) associated with the `<select>` via `htmlFor`. |
| Page title | `/projects/{id}` placeholder sets `document.title = "{name} — AgenticApps Dashboard"` (project name from router params; falls back to project id if name not yet loaded). Home page: `document.title = "AgenticApps Dashboard"`. |
| Skip link | Phase 2's "Skip to main content" link in `AppShell` remains in place. |
| Long-press | No accessible alternative needed for long-press because the same action is always available via the always-visible kebab button. |

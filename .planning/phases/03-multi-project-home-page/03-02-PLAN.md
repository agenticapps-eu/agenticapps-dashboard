Output: A `HomeLayout` component that, when wrapped around the home-route content, expands to `max-w-5xl`; all other routes (onboarding, pair, settings, help, projects/$id) inherit the existing AppShell `max-w-3xl`.
</objective>

**Approach selected:** HomeLayout wrapper inside the Outlet (Approach B from RESEARCH Pattern 18). Rationale:
1. Zero churn to AppShell — Phase 2 routes and tests untouched.
2. Wraps home route content; other routes opt out by not using it.
3. Tailwind v4 width arithmetic via `-mx-(*)` to break out of `max-w-3xl` constraint and re-apply `max-w-5xl`. Specifically: `<div class="-mx-[calc((100vw-768px)/2)] sm:mx-0 ... mx-auto max-w-5xl">` — but a simpler approach is to remove the AppShell width constraint and re-apply it inside HomeLayout. We'll use the cleanest option: AppShell accepts an optional `mainClassName` override, defaulting to `max-w-3xl`; HomeLayout is unused, instead the home route's lazy component wraps its content in a div with `-mt-8 -mx-6 md:-mx-8 px-6 md:px-8 max-w-5xl mx-auto`. **DECISION:** ship the AppShell `mainClassName` prop approach because it's simpler and more honest about where the constraint lives.

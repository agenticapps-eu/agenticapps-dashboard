### Gaps Summary

Two root-cause gaps block phase goal achievement:

**Gap 1 — Settings and Help routes missing PageHeader:** Plan 05.1-05 added PageHeader with a V2 flag guard, then Plan 06 was supposed to strip the flag and keep PageHeader unconditional. However, Plan 05's worktree changes were never merged to the `phase-05.1-ui-redesign` branch. When Plan 06 ran, it found settings.lazy.tsx in its Phase 2 state (no PageHeader) and skipped it. The routes are correctly wired under AppShellV2 but render without the per-route page header that UI-SPEC §7 + plan must_haves require.

**Gap 2 — Alias migration incomplete (14 component files + 2 route files with broken CSS vars):** Plan 05.1-05 performed a batch migration of all legacy `[--*]` CSS variable references to Tailwind 4 namespaced utilities (bg-card-bg, text-text-primary, etc.) via a Python batch script in commit 9f9545d. This commit was executed in a git worktree that was never merged back to the `phase-05.1-ui-redesign` branch. The Plan 05 docs commit (6b256b8) only added the SUMMARY.md/ROADMAP.md/STATE.md changes. Wave 6 (79e9b02) then deleted the global.css alias block (correctly) without first confirming that the component migration had landed on the branch. Result: 14 component files + 2 route files now reference undefined CSS custom properties at runtime. The automated invariant tests (tokenSourceOfTruth.test, noOrange.test) do not catch this because they check for hex literals and orange-family strings respectively — not for CSS variable reference validity.

These two gaps share the same root cause: Plan 05 worktree changes were not merged before Wave 6 executed the alias block deletion.

# Phase 05.1 Plan 01: Design Tokens + UI Primitives + Invariants Summary

**One-liner:** Tailwind-4 token system with Inter Variable font, 7 no-hex UI primitives, and two AC-04/AC-05 invariant tests enforcing the locked purple palette.

## What Was Built

### Token Foundation (Task 1)

`packages/spa/src/styles/tokens.css` is the new single source of truth. It uses Tailwind 4's `@theme` directive so utilities like `bg-card-bg`, `text-text-primary`, `shadow-card`, `rounded-card` are auto-generated from the locked values — no hex literals needed in component code.

### Backwards-Compat Aliases (Wave 5 deletion plan)

`global.css` now has a `:root` block mapping the 16 legacy token names to the new `--color-*` tokens. This keeps all 13 existing panels rendering unchanged through Waves 1-4:

```
--bg → var(--color-app-bg)
--surface → var(--color-card-bg)
--surface-elevated → var(--color-card-bg-hover)
--border → var(--color-border-subtle)
--border-strong → var(--color-border-subtle)
--text → var(--color-text-primary)
--text-muted → var(--color-text-secondary)
--text-subtle → var(--color-text-tertiary)
--accent → var(--color-accent)
--accent-hover → var(--color-accent-hover)
--accent-fg → #FFFFFF  (literal OK here — the alias block is in global.css, not a component)
--danger → var(--color-status-error)
--danger-surface → rgb(181 61 61 / 0.06)
--warning → var(--color-status-warning)
--warning-surface → rgb(194 128 43 / 0.06)
--success → var(--color-status-success)
--ring → var(--color-accent)
```

**Wave 5 (plan 05.1-06) deletes these aliases** — at that point every consumer will have been migrated to the namespaced utilities.

### Dark Theme (OQ-5 preserved)

The legacy `.dark { }` block from prior phases is preserved verbatim in `global.css`. Dark-mode users keep the Phase 2-5 palette until Phase 6 ships dedicated dark tokens.

### Pitfalls Addressed

- **Pitfall 1 (token namespacing):** All tokens use `--color-*`, `--font-*`, `--shadow-*`, `--radius-*`, `--text-*` prefixes so Tailwind 4 auto-generates the correct utilities.
- **Pitfall 3 (Inter font family name):** `--font-sans` uses `'Inter Variable'` (with space) — the exact name fontsource registers for the variable family.
- **Pitfall 5 (import order):** `import '@fontsource-variable/inter'` is the first line of `main.tsx`, before `'./styles/global.css'`.
- **Pitfall 7 (aria-labelledby contract):** `Card` accepts `ariaLabelledBy` prop mirroring `PanelContainer`'s pattern — enables wrapper-only swap in Waves 2-3 without rewriting test assertions.

### 7 UI Primitives (Task 2)

All primitives use Tailwind-4 namespaced utilities. No hex literals. No `cn()`/clsx/CVA. No transitions.

| Primitive | Key classes | Notes |
|---|---|---|
| `Card` | `rounded-card bg-card-bg p-6 shadow-card` | `ariaLabelledBy` prop wires `aria-labelledby`; same a11y contract as PanelContainer |
| `CardHeader` | `text-lg font-semibold text-text-primary` | `titleId` pairs with Card's `ariaLabelledBy`; action wins over helper |
| `EmptyState` | `text-lg font-semibold` / `text-base text-text-secondary` | Panel-specific content only (D-4-14) |
| `Pill` | `rounded-md px-2 py-0.5 text-xs font-medium` + VARIANT_CLASSES | 5 variants via inline lookup |
| `StatusPill` | `bg-accent-bg text-accent` (accent) or neutral | label · value separator pattern for TopBar |
| `MetricNumeric` | `text-3xl font-semibold tabular-nums` | tabular-nums for stable-width metrics |
| `KbdHint` | `font-mono text-xs border` | `aria-hidden="true"`; decorative ⌘K chip |

### Invariant Tests (Tasks 1 + 3)

- **`noOrange.test.ts` (AC-04):** Scans all `src/` files (excl. test files) for `#FFA500`, `#FF8800`, `#FFB347`, and `#0073EC`. Confirms `#d97706` appears only in `global.css` (the legacy alias block). 3 tests.
- **`tokenSourceOfTruth.test.ts` (AC-05):** Scans `src/components/**` production files for any hex literal (`/#[0-9a-fA-F]{3,8}\b/`). Fails if any hex appears outside `tokens.css`. Positive control confirms `#6B46C1` in `tokens.css`. 3 tests.

Both test files use `// @vitest-environment node` to run in Node (not jsdom) so `process.cwd()` resolves the file paths reliably.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 2 - Missing critical functionality] TypeScript module declaration for @fontsource-variable/inter**
- **Found during:** Task 1 typecheck
- **Issue:** `tsc --noEmit` raised `TS2882: Cannot find module or type declarations for side-effect import of '@fontsource-variable/inter'`
- **Fix:** Added `declare module '@fontsource-variable/inter' {}` to `src/vite-env.d.ts`
- **Files modified:** `packages/spa/src/vite-env.d.ts`
- **Commit:** 86e8c88

**2. [Rule 3 - Blocking issue] import.meta.url not file:// in jsdom environment**
- **Found during:** Task 1 (noOrange.test.ts) initial run
- **Issue:** The plan's template used `fileURLToPath(new URL('../', import.meta.url))` but jsdom sets `import.meta.url` to a non-file URL, causing `TypeError: The URL must be of scheme file`
- **Fix:** Used `// @vitest-environment node` comment + `resolve(process.cwd(), 'src')` instead
- **Files modified:** `packages/spa/src/styles/noOrange.test.ts`, `packages/spa/src/styles/tokenSourceOfTruth.test.ts`
- **Commit:** 86e8c88 (noOrange), 2e16486 (tokenSourceOfTruth)

**3. [Rule 1 - Import order lint] noOrange.test.ts and tokenSourceOfTruth.test.ts import ordering**
- **Found during:** Task 3 (pnpm lint)
- **Issue:** `vitest` import was placed before `node:fs`/`node:path` imports, violating `import/order` rule (node: imports must come first)
- **Fix:** Moved `import { describe, it, expect } from 'vitest'` after node: imports in both files
- **Files modified:** Both styles test files
- **Commit:** 2e16486

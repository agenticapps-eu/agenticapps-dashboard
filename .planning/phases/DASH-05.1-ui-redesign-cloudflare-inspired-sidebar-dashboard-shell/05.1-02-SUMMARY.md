## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Fixed addChildren mutation collision in router.tsx**
- **Found during:** Task 2
- **Issue:** Both legacy and V2 route trees used the same `rootRoute` instance. `addChildren()` mutates the root and returns `this` — V2's `addChildren` call always overwrote legacy, meaning `_appshell` was always in `routesById` regardless of flag, and e2e tests for `/projects/$projectId` matched the wrong path.
- **Fix:** Declared `legacyRootRoute` + `legacyRouter` at module scope (for TypeScript Register inference); `v2RootRoute` created inside `if (useV2)` block so the two trees never share a root. `Register` interface always uses `typeof legacyRouter` so all route paths are statically known.
- **Files modified:** `packages/spa/src/router.tsx`
- **Commit:** 003ceba

**2. [Rule 1 - Bug] Removed unused Outlet import from router.test.tsx**
- **Found during:** Task 2 lint run
- **Issue:** `import { Outlet } from '@tanstack/react-router'` was present in router.test.tsx but not used, causing lint error `@typescript-eslint/no-unused-vars`
- **Fix:** Removed the unused import
- **Files modified:** `packages/spa/src/router.test.tsx`
- **Commit:** 003ceba

**3. [Rule 1 - Bug] Fixed AV1 test: aside has ARIA role complementary, not navigation**
- **Found during:** Task 2 TDD GREEN phase
- **Issue:** Test used `getByRole('navigation', { name: /primary navigation/i })` but `<aside>` carries the `complementary` ARIA role, not `navigation`. The `<nav>` inside has the navigation role.
- **Fix:** Changed assertion to `container.querySelector('aside[aria-label="Primary navigation"]')`
- **Files modified:** `packages/spa/src/components/AppShellV2.test.tsx`
- **Commit:** 003ceba

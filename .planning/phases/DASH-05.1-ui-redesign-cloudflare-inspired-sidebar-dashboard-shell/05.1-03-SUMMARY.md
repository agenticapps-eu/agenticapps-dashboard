# Phase 05.1 Plan 03: Home Page Migration + Repalette Summary

**One-liner:** MultiProjectHome wraps in PageHeader (V2-gated via VITE_APPSHELL_V2) using useLastRefresh for the helper text; HomeToolbar, ProjectCard, and RegisterButtonCard fully migrated from legacy `[--*]` aliases to namespaced `bg-card-bg / text-text-primary / bg-accent / border-border-subtle` Tailwind-4 utilities.

## What Was Built

### Task 1 — PageHeader integration in MultiProjectHome (V2-gated)

`MultiProjectHome.tsx` now reads `import.meta.env.VITE_APPSHELL_V2 === '1'` at the top of its function body. When true:

- Renders `<PageHeader title="Projects" helper={headerHelper} />` before the toolbar/grid
- `headerHelper` is derived from `useLastRefresh()`: `"{N} projects · {refreshLabel}"` when count is available, empty string while loading
- Drops the inner `px-6 py-8 md:px-8` wrapper in V2 mode (AppShellV2 already provides `p-6` outer padding)
- Legacy mode (`flag=OFF`): renders identically to pre-Phase-5.1 (HomeLayout wraps, no PageHeader)

Empty-state block repaletted from `border-[--border] bg-[--surface]` → `border-border-subtle bg-card-bg rounded-card shadow-card` with `text-text-primary` / `text-text-secondary` text.

### V2 Gate Pattern

```tsx
const useV2 = import.meta.env.VITE_APPSHELL_V2 === '1'
const lastRefresh = useLastRefresh()
const headerHelper =
  lastRefresh.count !== null
    ? `${lastRefresh.count} ${lastRefresh.count === 1 ? 'project' : 'projects'} · ${lastRefresh.refreshLabel ?? ''}`
    : ''

return (
  <HomeLayout>
    {useV2 && <PageHeader title="Projects" helper={headerHelper} />}
    <div className={useV2 ? '' : 'px-6 py-8 md:px-8'}>
      {/* existing toolbar + grid + modals */}
    </div>
  </HomeLayout>
)
```

Reading `import.meta.env` inside the function body (not at module scope) enables `vi.stubEnv('VITE_APPSHELL_V2', '1')` in tests without requiring `vi.resetModules()` + dynamic import — simpler test scaffolding.

### Task 2 — Token translation in HomeToolbar, ProjectCard, RegisterButtonCard

Complete token translation applied to all three files. Zero legacy `[--*]` arbitrary-value classes remain.

#### Token translation table applied

| Legacy alias | Namespaced Tailwind-4 utility |
|---|---|
| `bg-[--surface]` / `bg-[--surface-elevated]` | `bg-card-bg` / `bg-card-bg-hover` |
| `border-[--border]` / `border-[--border-strong]` | `border-border-subtle` |
| `text-[--text]` | `text-text-primary` |
| `text-[--text-muted]` | `text-text-secondary` |
| `text-[--text-subtle]` | `text-text-tertiary` |
| `bg-[--accent]` | `bg-accent` |
| `text-[--accent]` | `text-accent` |
| `border-[--accent]` | `border-accent` |
| `text-[--accent-fg]` | `text-white` |
| `focus-visible:ring-[--ring]` | `focus-visible:ring-accent` |
| `focus-visible:ring-offset-[--bg]` | `focus-visible:ring-offset-app-bg` |
| `text-[--danger]` | `text-status-error` |
| `text-[--warning]` | `text-status-warning` |
| `rounded-md` (card root) | `rounded-card` |

**HomeToolbar:** CHIP_SELECTED uses `bg-accent text-white border-accent`; CHIP_UNSELECTED uses `bg-card-bg text-text-primary border-border-subtle hover:bg-card-bg-hover`. Search input and sort select use `bg-card-bg-hover border-border-subtle text-text-primary`. Icons use `text-text-secondary`.

**ProjectCard:** Card root uses `bg-card-bg border-border-subtle rounded-card shadow-card` (UI-SPEC §8 card anatomy). Kebab uses `text-text-secondary hover:bg-card-bg-hover hover:text-text-primary`. All inline text spans migrated.

**RegisterButtonCard:** `border-2 border-dashed border-border-subtle bg-card-bg rounded-card text-text-secondary hover:border-accent hover:text-accent`.

### HomeLayout: still active, not deleted

HomeLayout still wraps in both V2 and legacy modes. In V2 mode, `setAppShellWidth('max-w-5xl')` still fires — harmless since AppShellV2 has its own fixed grid. Wave 5 (plan 05.1-06) deletes HomeLayout after all consumers migrate.

**Hard rules:**
- **No orange anywhere.** Cloudflare's identity color, not ours.
- **Accent purple appears only on:** logo glyph, active sidebar pill, active tab indicator, primary action button bg, key metric numerals (one per panel), focus ring.
- **No gradient backgrounds.** Solid colors only.
- **No more than three weights of border in a single view.** (Subtle, none, focus-ring is the ladder.)

## 3. Typography Scale

Single family: **Inter** (variable). Fallback stack: `Inter, ui-sans-serif, system-ui, -apple-system, sans-serif`. **No mono** in v1 — numerics are tabular Inter (`font-feature-settings: 'tnum' 1`).

```css
/* Type scale — packages/spa/src/styles/tokens.css */
:root {
  --font-sans: 'Inter', ui-sans-serif, system-ui, -apple-system, sans-serif;

  --text-xs:   11px;  /* uppercase section labels, tracking-wider */
  --text-sm:   12px;  /* card helper, captions */
  --text-base: 13px;  /* body, card content default */
  --text-md:   14px;  /* sidebar items, page subtitle */
  --text-lg:   16px;  /* card headers */
  --text-xl:   20px;  /* page title section labels */
  --text-2xl:  24px;  /* page title */
  --text-3xl:  32px;  /* big metric numerals (e.g., Skill Health 92) */

  --leading-tight:  1.2;
  --leading-normal: 1.5;
  --leading-relaxed: 1.65;

  --weight-regular:  400;
  --weight-medium:   500;
  --weight-semibold: 600;
}
```

**Tabular numerics:** apply `font-feature-settings: 'tnum' 1` to any element rendering metrics, scores, timestamps, counts.

**Tracking:** uppercase section labels use `letter-spacing: 0.06em`; everything else default.

## 4. Spacing & Radius

Use Tailwind's default 4px scale. Specific commitments:

- **Card padding:** 20-24px (`p-5` to `p-6`).
- **Card gap (vertical, in a column):** 24px (`gap-6`).
- **Column gap (horizontal):** 24px (`gap-x-6`).
- **Sidebar item padding:** 8px vertical, 12px horizontal (`py-2 px-3`).
- **Outer page padding:** 24px (`p-6`).
- **Card radius:** `--radius-card` = 12px. Corners are soft, not sharp.
- **Pill radius:** 6px (slightly less than card; intentional).
- **Input radius:** 6px.
- **Avatar radius:** 50% (full circle).

## 7. Page Header (per-route)

Sits inside the main content area, above the route's content. Different per route but always:

```
┌─────────────────────────────────────────────────────────────┐
│  Page Title                              [actions on right] │
│  helper text (timestamp, last-refresh, etc.)                 │
└─────────────────────────────────────────────────────────────┘
```

- **Title:** 24px semibold (`--text-2xl`).
- **Helper:** 13px tertiary text (`--text-tertiary`).
- **Actions:** ghost or primary buttons, right-aligned.
- **Bottom margin:** 24px before route content begins.

**Component:** `PageHeader { title, helper, actions, children? }`.

## 8. Card Anatomy

```
┌──────────────────────────────────────────┐
│ [icon] Card Header           [optional] │  ← lg semibold + sm helper-right
│                                         │
│  Card body content                      │  ← base body
│                                         │
└──────────────────────────────────────────┘
```

- **Background:** `--bg-card`, no border.
- **Shadow:** `--shadow-card` (subtle).
- **Radius:** 12px (`--radius-card`).
- **Padding:** 20-24px.
- **Header row:** 16px line icon (lucide-react) + semibold label, optional small helper text right-aligned.
- **Body:** 13px regular, 1.5 line-height.
- **Empty state:** D-4-14 per-panel discipline preserved. Each empty state is hand-written, not a generic "no data".

**Components to extract / consolidate:**
- `Card` — root surface.
- `CardHeader { icon, label, helper, action? }` — replaces ad-hoc panel headers.
- `EmptyState { icon, title, body, action? }` — consolidated empty surface.
- `Pill { variant: 'neutral' | 'accent' | 'success' | 'warn' | 'error', children }`.
- `MetricNumeric { value, suffix?, label? }` — big numeral with tabular-nums.

## 10. Migration Plan (sequencing for the planner)

This is a **shell + token migration**, not a content rewrite. Existing panel components (HookFirings, PhaseProgress, SkillHealth, etc.) keep their data wiring; only their wrapper changes from "ad-hoc div with bg-white p-4" to `<Card><CardHeader …/>…</Card>`.

**Suggested wave order** (planner has discretion to re-sequence):

- **Wave 0 — Tokens + primitives.** Land `tokens.css`, `Card`, `CardHeader`, `EmptyState`, `Pill`, `StatusPill`, `MetricNumeric`, `KbdHint`. No route impact yet.
- **Wave 1 — Shell.** Land `Sidebar*`, `TopBar`, `Breadcrumb`, `PageHeader`, `AppShellV2`. Mount AppShellV2 behind a dev-only env flag (`VITE_APPSHELL_V2=1`). Old AppShell still default.
- **Wave 2 — Migrate `/`.** Wrap `MultiProjectHome` content in PageHeader + repalette. Swap ProjectCard styling.
- **Wave 3 — Migrate `/projects/:id`.** Wrap `SingleProjectView` content. Migrate each Phase 5 panel to use the new `Card` primitive.
- **Wave 4 — Migrate `/settings` + `/help`.** Repalette + use PageHeader.
- **Wave 5 — Switch flag default.** Make AppShellV2 the default; old AppShell removed in the same commit. Final regression sweep.

Each wave ships independently green (typecheck + tests + visual smoke on the dev server).

## 11. Acceptance Criteria

The phase is done when:

1. **All paired routes (`/`, `/projects/:id`, `/settings`, `/help`) render under AppShellV2** with the warm-paper sidebar and the locked design tokens.
2. **`/onboarding` and `/pair` are unchanged** (still standalone shells).
3. **Every Phase 5 panel still works** — same data, same behavior, no regressions in the existing 1117-test suite.
4. **No orange appears anywhere** in the SPA's chrome or panels.
5. **The token file is the single source of truth** — no hardcoded colors in component files.
6. **`impeccable:critique` ≥ 90** on `/` and `/projects/:id` at desktop 1440x900 (preview of the Phase 6 gate; not strict enforcement here, but a strong signal).
7. **Component tests pass** for each new UI primitive (Sidebar, Card, etc.).
8. **Visual smoke check** in the live dev server (Playwright screenshot of `/projects/agenticapps-dashboard`) is committed under `.planning/phases/DASH-05.1-…/refs/after-shell.png`.

## 13. Implementation Notes for `frontend-design`

When `frontend-design` is invoked to implement this spec:

1. **Read tokens.css first.** All colors, typography, and spacing values come from there. Do not hardcode hex values in components.
2. **Use lucide-react icons exclusively.** No emoji, no custom SVG except the app logo glyph.
3. **Tabular numerics on every metric.** Wrap big numerals in `<span class="tabular-nums">` (Tailwind utility) or use `MetricNumeric`.
4. **Avoid skeleton-shimmer.** Loading states use static placeholders ("Loading…" text or a single static spinner glyph).
5. **Anti-slop checklist before commit:** no animations, no gradients, no shadows beyond `--shadow-card`, no rounded-full on anything except avatars + status dots.
6. **Real Phase 5 data only.** Variant E is NOT what we're shipping — that was a generative artifact. Implementer pulls actual data via existing TanStack Query hooks.

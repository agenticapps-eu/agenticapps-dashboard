## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] E2E mock REGISTRY_RESPONSE missing status field**
- **Found during:** Task 1 — `pnpm -r test` after flag removal
- **Issue:** `projects-detail-e2e.test.tsx` REGISTRY_RESPONSE mock lacked the `status` field. The bug was latent while AppShell was the default shell; once AppShellV2 became unconditional, `Sidebar` always runs `useRegistryList()` and accesses `p.status.reachable` — causing `Cannot read properties of undefined (reading 'reachable')`
- **Fix:** Added `status: { reachable: true, currentPhase: '04-single-project-view', lastCommitAt: '...' }` to the mock; also corrected `path` → `root` field name
- **Files modified:** `packages/spa/src/__tests__/projects-detail-e2e.test.tsx`
- **Commit:** `b337316`

**2. [Rule 1 - Bug] E2E1 test asserted ProjectHeader breadcrumb nav (deleted component)**
- **Found during:** Task 1 test run
- **Issue:** `E2E1` asserted `screen.getByRole('navigation', { name: 'Project breadcrumb' })` — ProjectHeader's nav element. Once ProjectHeader was deleted, this test would always fail.
- **Fix:** Updated E2E1 to assert `screen.getByRole('heading', { level: 1, name: 'acme' })` — the PageHeader h1 that replaced it
- **Files modified:** `packages/spa/src/__tests__/projects-detail-e2e.test.tsx`
- **Commit:** `b337316`

**3. [Rule 1 - Bug] useParams from: '/_appshell/projects/$projectId' broke test fixtures**
- **Found during:** Task 1 — route test failures
- **Issue:** Initial projects.$projectId route used `useParams({ from: '/_appshell/projects/$projectId' })`. Test fixtures build minimal routers with the route at root level (path `/projects/$projectId`), so the `_appshell` prefix didn't exist and TanStack Router threw `Invariant failed: Could not find an active match`
- **Fix:** Changed to `useParams({ strict: false })` which works for both the real V2 tree and test fixtures
- **Files modified:** `packages/spa/src/routes/projects.$projectId.lazy.tsx`
- **Commit:** `b337316`

**4. [Rule 1 - Bug] Unused `afterEach` import in MultiProjectHome.test.tsx**
- **Found during:** Task 2 — `pnpm lint` returned 1 error
- **Issue:** Removing the `VITE_APPSHELL_V2` describe block (which had `afterEach(() => vi.unstubAllEnvs())`) left `afterEach` imported but unused — `@typescript-eslint/no-unused-vars` error
- **Fix:** Removed `afterEach` from the vitest import
- **Files modified:** `packages/spa/src/components/MultiProjectHome.test.tsx`
- **Commit:** `79e9b02`

**5. [Rule 2 - Route TDD test IDs] Router test IDs use /_appshell/ prefix**
- **Found during:** Task 1 GREEN phase — RT1 failed expecting route id `/`
- **Issue:** TanStack Router's pathless layout routes prefix all child IDs with the layout ID. The index route under `_appshell` has id `/_appshell/` not `/`
- **Fix:** Updated router.test.tsx assertions to match actual IDs (`/_appshell/`, `/_appshell/settings`, etc.)
- **Files modified:** `packages/spa/src/router.test.tsx`
- **Commit:** `b337316`

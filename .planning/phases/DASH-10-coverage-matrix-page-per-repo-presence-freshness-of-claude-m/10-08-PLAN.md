    # ADR 0021: Coverage Matrix Page

    ## Status
    Accepted (Phase 10 of agenticapps-dashboard).

    ## Context

    The AgenticApps Superpowers + GSD + gstack workflow defines a knowledge layer per repo
    (CLAUDE.md), per family (wiki), globally (GitNexus index), and per workflow installation
    (skill version). Until Phase 10, there was NO surface answering "which repos are doing
    their knowledge-layer homework, and what needs attention?"

    ## Decision

    Ship a /coverage page in agenticapps-dashboard that scans every git repo under
    ~/Sourcecode/{agenticapps,factiv,neuroflash} (one level deep, excluding personal/shared/archive)
    and surfaces a 4-column presence/freshness matrix.

    ### Columns (4)
    1. CLAUDE.md (or AGENTS.md fallback) — exists / missing
    2. GitNexus indexed — fresh ≤ 14 days / stale > 14 days / missing / not-applicable (no ~/.gitnexus)
    3. Wiki linked — both .wiki-compiler.json source reference AND .knowledge/wiki/.compile-state.json ≤ 7 days
    4. Workflow version — matches highest migration's to_version / behind / ahead / missing-skill / version-unknown

    ### Layout
    Grouped sections per family with sticky family-header aggregate counts (preserves ~/Sourcecode/CLAUDE.md
    family-boundary contract while keeping cross-family visibility on one page).

    ### Override surface
    Inline `⚠ N override` chip per row when `<repo>/.planning/phases/*/multi-ai-review-skipped`
    sentinels exist. Sentinels are user-controlled audit signals (ADR 0018). The GSD_SKIP_REVIEWS=1
    env-var override is undetectable (no on-disk trace) — documented gap.

    ### Refresh semantics (D-10-02 + D-10-09)
    - GitNexus stale → daemon spawns `gitnexus analyze` (PATH-resolved binary, argv-array, NEVER `npx`)
    - Wiki stale → CLIPBOARD-ONLY (no headless `/wiki-compile` runner exists)
    - CLAUDE.md missing → help-link (no daemon action)
    - Workflow version mismatch → clipboard copy of `claude /update-agenticapps-workflow`

    ### Cache (D-10-01)
    30s daemon-side memo cache (Map<'all', {value, expiresAt}>); cleared on POST /coverage/refresh.

    ## Consequences

    Positive:
    - Single surface for cross-family knowledge-layer health
    - GAPS are surfaced (e.g. neuroflash has 33 repos but .wiki-compiler.json references only 8)
    - Self-updating workflow head (migration `to_version` is single source of truth)

    Negative:
    - Wiki refresh requires user terminal interaction (clipboard) — not fully automated in v1
    - GSD_SKIP_REVIEWS env-var override is invisible to the coverage page
    - 30s cache means freshness can be up to 30s stale (acceptable — coverage data churns slowly)

    Mitigations:
    - Future migration may add a headless wiki-compile runner; v1 documents the limitation in UI subtext
    - Audit on env-var overrides relies on user discipline + git log conventions

    ## Alternatives Considered

    [Reference 10-CONTEXT.md "Deferred Ideas" — explain why each alternative was rejected]
    - Cross-family aggregate health-score % hero number → deferred to v1.2
    - Per-repo drill-down detail page → row is self-contained in v1
    - AgentLinter as 5th column → defer; AgentLinter already in per-project view
    - chokidar background watcher → rejected per D-10-01 (pull-only)

    ## Related Decisions
    - ADR 0018 — multi-ai-plan-review-enforcement (sentinel override data source)
    - ADR 0019 — llm-wiki-compiler-integration (Wiki column data source)
    - ADR 0020 — gitnexus-code-graph-integration (GitNexus column — note: 0020's verify script has a `.repos | length` bug corrected in migration 0008)

import { describe, it, expect } from 'vitest'
import { writeFileSync, mkdtempSync, rmSync } from 'node:fs'
import { tmpdir } from 'node:os'
import { join } from 'node:path'
import { parseFrontmatter } from '../src/lib/skillsScan.js'

// CODEX MED-17: fixture test ALWAYS runs (no skip-if-missing). Protects COV-12 in CI.
// The expected frontmatter shape is locked here; if the upstream migration 0008 file
// ships with a different shape, the cross-repo smoke test (sibling file) catches it.
const DELIM = '---'
const FIXTURE_CONTENT = [
  DELIM,
  'id: 0008',
  'slug: coverage-matrix-page',
  'title: Coverage Matrix Page',
  'from_version: 1.7.0',
  'to_version: 1.8.0',
  'type: workflow-surface',
  DELIM,
  '',
  '# Migration 0008',
  '',
  'clipboard-only wiki refresh (D-10-09).',
  "jq 'length' ~/.gitnexus/registry.json — top-level array.",
].join('\n')

describe('migration 0008 fixture (CODEX MED-17 — CI-resident)', () => {
  let dir: string
  let path: string

  beforeEach(() => {
    dir = mkdtempSync(join(tmpdir(), 'mig0008-fixture-'))
    path = join(dir, '0008-coverage-matrix-page.md')
    writeFileSync(path, FIXTURE_CONTENT, 'utf8')
  })

  afterEach(() => {
    rmSync(dir, { recursive: true, force: true })
  })

  it('parses expected frontmatter shape', () => {
    const fm = parseFrontmatter(path)
    expect(fm).not.toBeNull()
    expect(fm).toMatchObject({
      id: '0008',
      slug: 'coverage-matrix-page',
      from_version: '1.7.0',
      to_version: '1.8.0',
      type: 'workflow-surface',
    })
  })

  it('contains the jq correction language (Pitfall 7)', () => {
    expect(FIXTURE_CONTENT).toMatch(/jq 'length'/)
    expect(FIXTURE_CONTENT).toMatch(/top-level array/i)
  })

  it('documents clipboard-only wiki refresh (D-10-09)', () => {
    expect(FIXTURE_CONTENT).toMatch(/clipboard.?only/i)
    expect(FIXTURE_CONTENT).toMatch(/wiki/i)
  })
})

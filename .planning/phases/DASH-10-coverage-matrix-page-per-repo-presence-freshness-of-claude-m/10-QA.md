## Issues found

### Minor (Nice to Have)

**ISSUE-001 — Keyboard-hints popup overlaps with "Refresh 0 stale" button on first render.**
- **Severity:** low (cosmetic / first-impression).
- **Where:** Top-right corner of `/coverage` on first visit (before user clicks "Got it" anywhere in the app).
- **What I saw:** `./screenshots/01-coverage-initial.png` shows the "Press R to refresh, ? for help, / to search" hint card rendered behind the disabled "Refresh 0 stale" button. The hint card auto-dismisses on first user interaction.
- **Why it matters:** Slightly confusing on first impression but resolves itself the moment the user clicks anything.
- **Suggested fix:** Either z-index the hint above the page action, or anchor it lower so it doesn't overlap the page header's `actions` slot. Not a Phase 10 regression — the keyboard-hints popup is a global Phase 06 affordance.

**ISSUE-002 — Empty family sections still render their headers + empty tables when filter excludes all rows.**
- **Severity:** low (UX).
- **Where:** `/coverage?status=stale&q=workflow` — `factiv · 0 repos` and `neuroflash · 0 repos` headers render with empty table bodies below them.
- **What I saw:** `./screenshots/04-search-workflow.png` — two large empty sections under the matching agenticapps results.
- **Why it matters:** Slightly noisy when filters exclude entire families.
- **Suggested fix:** When `rows.length === 0` for a family AND a non-default filter is active, either hide the section entirely or render as a one-line "neuroflash · 0 matching" without the empty table. Not blocking — the empty headers do correctly reflect that the filter applied; just visually heavier than necessary.

### Not found

- No console errors of any kind.
- No broken links.
- No layout breakage at tablet or desktop widths.
- No schema-drift surfaced (the daemon's outbound parse held throughout the session).
- No `absPath` leaked into the DOM (CODEX HIGH-1).
- No flicker, no double-fetch, no stale UI state after filter/search/collapse interactions.

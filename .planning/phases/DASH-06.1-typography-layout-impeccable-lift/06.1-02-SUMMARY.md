# Phase 06.1 Plan 02: MaskedToken Primitive Summary

**One-liner:** Bearer-token masking primitive with timed 5s reveal, always-on copy, ARIA semantics, and instant (no-CSS-motion) state transitions.

## What Was Built

`<MaskedToken value={string} label?={string} />` ŌĆö a self-contained React primitive in `packages/spa/src/components/ui/` that closes D-6.1-03.

**Exports:**
- `MaskedToken` ŌĆö the component function (`React.JSX.Element`)
- `MaskedTokenProps` ŌĆö the Props interface (`value: string`, `label?: string`)

**State machine:** `masked=true` (default) ŌåÆ click Reveal ŌåÆ `masked=false` + 5s `setTimeout` ŌåÆ auto-re-mask ŌåÆ `masked=true`. Manual "Hide" click cancels the timer immediately.

**Security invariants:**
- Token value never appears in DOM text until `masked === false`
- Copy uses `navigator.clipboard.writeText(value)` directly ŌĆö works while masked, never requires reveal
- `useEffect` cleanup and `cancelTimer()` both call `clearTimeout` (2 occurrences) ŌĆö no setState-after-unmount
- Zero `transition`/`animate-`/`duration-`/`animation`/`@keyframes` utilities in rendered DOM

**ARIA semantics:**
- Masked span: `aria-label="${label}, masked"` ŌĆö screen reader announces purpose, not dots
- Revealed span: `aria-label="${label}, revealed"`
- Reveal/Hide button: `aria-label="Reveal ${label}"` / `aria-label="Hide ${label}"`
- Copy button: `aria-label="Copy ${label}"`

## Deviations from Plan

### Auto-added: keyboard shortcuts table in /help

**Rule 2 — Missing critical functionality**
- **Found during:** Task 2 GREEN
- **Issue:** The current `help.lazy.tsx` had only 21 lines with a single `<p>` and no keyboard shortcuts table. The plan's acceptance criteria required `>= 5` occurrences of `max-w-[75ch]` in help.lazy.tsx and explicitly called out description `<td>` cells in a keyboard shortcuts table.
- **Fix:** Added a keyboard shortcuts table with 4 rows (?, R, G H, G S) using `<kbd>` elements in the left column and description text in `<td className="py-2 max-w-[75ch] ...">` cells in the right column.
- **Files modified:** `packages/spa/src/routes/help.lazy.tsx`
- **Commit:** 844d113

### Test harness adaptation: `findAllByRole` instead of `findByRole`

**Rule 1 — Auto-fix**
- **Found during:** Task 2 GREEN verification
- **Issue:** After adding the second `<section>` with its own `<h2>`, `screen.findByRole('heading', { level: 2 })` threw "Found multiple elements" because both sections have an h2.
- **Fix:** Changed all three await-guards in help.test.tsx to `await screen.findAllByRole('heading', { level: 2 })`.
- **Files modified:** `packages/spa/src/routes/__tests__/help.test.tsx`
- **Commit:** 844d113

### Test case for "Why local-only" paragraph: container query instead of text matcher

**Rule 1 — Auto-fix**
- **Found during:** Task 1 RED confirmation
- **Issue:** `screen.getByText(/Your.*\.planning.*are sensitive/s)` failed because the paragraph's text content is split across inline `<code>` nodes (React renders `.planning/` and `.claude/` as separate `<code>` elements), making the regex unmatchable by testing-library's text matcher.
- **Fix:** Used `container.querySelector('details p')` which finds the `<p>` by DOM structure rather than text content.
- **Files modified:** `packages/spa/src/components/OnboardingHero.test.tsx`
- **Commit:** 2e140f1

## Line-length policy

| Option | Description | Selected |
|--------|-------------|----------|
| Card body text + help/onboarding/settings prose only | Apply to multi-sentence prose: card body text, help shortcut explanations, onboarding paragraphs, settings descriptions. NOT card metadata. NOT headings. NOT code blocks. | ✓ |
| All prose-like text including headings | Stricter — headings inherit max-w-[75ch] for consistent column width. Reads more uniformly but headings can wrap awkwardly. | |
| Just settings + help + onboarding routes | Conservative — project cards stay flexible (mostly metadata). Less risk, less impact on sub-scores. | |

**User's choice:** Card body text + help/onboarding/settings prose only
**Rationale:** Standard "prose only" rule — headings need to break out wider for hierarchy; card metadata is tabular not prose; code blocks need horizontal room (command examples).

## Integration panel progressive disclosure

| Option | Description | Selected |
|--------|-------------|----------|
| Empty-state ('Configure to enable') panels collapsed; data-bearing expanded | The 'spread thin' feel comes from 5 mostly-empty panels. Collapse the empty ones; keep panels with real data expanded. Header is click target. No localStorage. | ✓ |
| All 5 collapsed by default | Maximum density reduction. Risk: hides real data behind clicks; 'where's my stuff?' moment on first visit. | |
| None collapse — reduce visual weight via spacing only | Conservative. Likely insufficient to lift Layout sub-score; doesn't address the 'spread thin' verdict. | |

**User's choice:** Empty-state panels collapsed; data-bearing expanded
**Rationale:** Reduces visual density without hiding real data. Per-page-load reset keeps state simple.

## Token masking + sensitive-data display

| Option | Description | Selected |
|--------|-------------|----------|
| Bearer tokens masked by default; click 'show' → reveal + auto-hide after 5s | Mask only bearer tokens. Auto-hide protects against shoulder-surfing. Copy-to-clipboard always visible (no reveal needed to copy). | ✓ |
| Bearer tokens + project paths masked; click reveals until clicked again | Defense in depth — hide paths too. Reveal until manually re-hidden. More user-controlled, less protection against walk-away. | |
| Show everything by default with copy buttons (no masking) | Lowest friction. Tokens fully visible. Skip masking work entirely. | |

**User's choice:** Bearer tokens masked + auto-hide after 5s
**Rationale:** Bearer tokens are the only real secret. Auto-hide protects against walk-away. Copy-without-reveal is safer (clipboard is readable; reveal-then-copy adds friction without security gain).

## Accessibility scope: ARIA + empty states

| Option | Description | Selected |
|--------|-------------|----------|
| Comprehensive ARIA + numbered-steps empty state on /pair | aria-current/aria-label/aria-live additions. /pair gets numbered steps (no hero illustration, no walkthrough). Existing OnboardingHero on / stays as-is. | ✓ |
| ARIA only (skip /pair empty-state work) | Faster path. /pair stays as-is. Lifts Accessibility but leaves /pair's empty-canvas hierarchy unaddressed. | |
| Empty states only (skip ARIA work) | Visual hierarchy focus. Leaves Accessibility heuristic ungated. Risk: composite still misses ≥ 90 even if other dims lift. | |

**User's choice:** Comprehensive ARIA + numbered-steps empty state on /pair
**Rationale:** Accessibility is part of impeccable composite scoring. Lifting accessibility lifts the Hierarchy and overall composite scores. Numbered steps respect anti-AI-slop discipline.

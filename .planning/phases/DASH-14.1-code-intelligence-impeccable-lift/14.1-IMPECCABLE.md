## What changed (TDD)

Five targeted fixes to `CodeIntelligencePage.tsx`, each addressing a named `14-IMPECCABLE.md` finding:

1. **Error recovery (H9 +2).** Error state now shows plain-language cause ("The dashboard agent could not be reached…") + a **Retry** button (`coverageQuery.refetch`), replacing a dead-end that printed raw `error.message`.
2. **Status communicates (H1 +1).** Fresh rows show a green **"current"** pill instead of a blank Status cell (the Jordan-persona "is status broken?" signal).
3. **Actions explains itself (H6 +1).** When no viewer link, the Actions cell shows **"Install viewer to open"** / "Viewer unavailable" instead of blank.
4. **Relative time (H2/H4).** "Last analyzed" uses `formatRelativeTime` ("9d ago"), matching the rest of the app and restoring row-order visibility that locale day-formatting hid.
5. **Header consistency (H4/H8 +1).** Dropped `uppercase tracking-wider` from the table headers — the one generic-SaaS tic `14-IMPECCABLE` flagged; headers now match the app's lowercase `text-xs text-text-tertiary` convention.

**TDD:** `test(14.1) RED` (4 new behaviours + updated date test) → `feat(14.1) GREEN`. Full spa suite **1211+ green**, typecheck + eslint clean.

## Re-critique caught a real regression (fixed)

The first GREEN wired the status pills to **non-existent tokens** (`bg-success-bg`/`bg-warning-bg`/`text-*-text`) that resolve to transparent — so "current"/"stale" rendered as undifferentiated plain text. The pre-existing stale pill had the same bug. Fixed to the canonical app pattern `bg-status-success/10 text-status-success` (and `-warning`), matching CoverageCell/OverrideChip/FamilyCard. **Live-verified:** current pill = green tint `rgba(49,118,88,0.1)` (text `rgb(46,125,91)`), stale pill = amber tint `rgba(186,128,39,0.1)` (text `rgb(194,128,43)`) — non-transparent and visually distinct. Added className-token test assertions so the transparent-token regression can't return (jsdom can't catch it via text content).

## Residual (non-blocking, documented)

- **[P2] Same-day ordering.** `formatRelativeTime` caps at day granularity, so repos analyzed on the same day still read identically ("4d ago"). Improves on the locale-date version (which collapsed ALL rows) but doesn't fully order same-day cohorts. A future enhancement could add hour granularity or a secondary sort key.
- **[Strategic, out of scope] "Does `/code-intelligence` earn its sidebar section?"** It remains a projection of data `/coverage` already shows; the unique value (a glimpse *into* a graph — node counts, clusters, thumbnail) is still absent. This lift makes the surface read as a finished product, but the strategic question is a roadmap item, not a polish fix.

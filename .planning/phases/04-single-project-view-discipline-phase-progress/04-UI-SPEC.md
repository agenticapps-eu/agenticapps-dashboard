## Spacing Scale

Tailwind v4 utilities map to 4px base. Phase 4 follows the inherited Phase 2/3 scale:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-text gaps (`gap-1`), badge inner padding (`px-1 py-0.5`) |
| sm | 8px | Panel header icon-to-title gap (`gap-2`), inline between sibling items (`gap-2`) |
| md | 16px | Panel internal padding (`p-4`), panel-to-panel column gap |
| lg | 24px | Column outer padding (`p-6` on panel containers, matching `DaemonUnreachableState` pattern) |
| xl | 32px | Vertical space between the header row and the column grid |
| 2xl | 48px | Not used at panel level — reserved for major page breaks |
| 3xl | 64px | Not used in Phase 4 |

Exceptions:
- Touch targets for the back-navigation link and any icon buttons: min 44×44px (per WCAG 2.5.5). Achieved with `h-11 w-11` or `p-3` padding expansion.
- `CodeBlock` copy button: 36×36px (`h-9 w-9`) — existing Phase 3 pattern; acceptable because it is paired with a large adjacent `<code>` target.

## Typography

Inherited from Phase 2 global.css. No new type roles in Phase 4.

| Role | Size | Weight | Line Height | Tailwind class |
|------|------|--------|-------------|----------------|
| Label | 14px (`text-sm`) | 400 regular | 1.4 | `text-sm leading-snug` |
| Body | 16px (`text-base`) | 400 regular | 1.5 | `text-base leading-relaxed` |
| Heading | 20px (`text-xl`) | 600 semibold | 1.3 | `text-xl font-semibold leading-snug` |
| Display | 28px (`text-[1.75rem]`) | 600 semibold | 1.2 | Not used in Phase 4 panels |

Phase 4-specific typography rules:
- **Panel title:** Heading role (20px/600) — matches `SchemaDriftState` `<h2>` pattern.
- **Monospace content** (commit SHAs, filenames, code blocks, copy-pasteable commands): `font-mono text-sm` (`--font-mono` stack, 14px). This covers: ExecutionTimeline commit subjects, PhaseProgress filenames, CommitmentBlock verbatim markdown, DISC-04 install command.
- **Severity badge text:** Label role (14px/400), all-caps transform via `uppercase tracking-wide text-xs`.
- **Timestamp / freshness labels:** Label role (14px) in `text-[--text-muted]`.
- **Empty-state body copy:** Body role (16px/400) in `text-[--text-muted]`.
- **Header breadcrumb:** Label role (14px/400), `text-[--text]` for active segment, `text-[--text-muted]` for separators and deemphasized segments.

## Layout Contract

### Page shell (`SingleProjectView`)

**Source decision:** D-4-09 (CONTEXT.md)

```
AppShell (sticky h-14 header + CommandPalette + RepairBanner)
└── main#main  [mx-auto w-full max-w-7xl px-6 py-8]
    └── SingleProjectView
        ├── ProjectHeader  [mb-6]  (see header spec below)
        └── grid  [grid-cols-[1fr_1.5fr] gap-6 items-start]
            ├── LeftColumn   (Discipline: CommitmentBlock, HookFirings, RationalizationFires)
            └── CenterColumn (Phase Progress: PhaseProgress, ExecutionTimeline, ReviewStatus,
                              SecurityStatus, VerificationStatus)
```

- `max-w-7xl` — the detail page uses more horizontal width than the 3xl default of non-home routes. Achieved via `HomeLayout`-style width override (`setAppShellWidth('max-w-7xl')` on mount, reset on unmount).
- Grid columns: `grid-template-columns: 1fr 1.5fr`. Phase 5 changes this to `1fr 1.5fr 1fr` by adding the right-column component and updating one class. No Phase 5 placeholder element or skeleton exists in Phase 4 DOM.
- Column gap: `gap-6` (24px). Panel-to-panel vertical gap within a column: `flex flex-col gap-4` (16px).
- **No right-column DOM element** in Phase 4. Grid renders with two columns. The right half of the screen simply shows `--bg`.

### ProjectHeader

**Source decision:** D-4-10 (CONTEXT.md)

Single row, `h-auto` (wraps on narrow viewports), no sticky behavior (the main `<Header>` already handles sticky).

Render: `← All Projects · {name}{(client) if client} · {branch ?? '(no branch)'} · phase {paddedPhase} — {status}`

- `← All Projects` — TanStack `<Link to="/">` with `ChevronLeft` (16px) icon from lucide-react. Keyboard: Tab-reachable, Enter/Space activates.
- Separator ` · ` — `aria-hidden="true"` `<span>` in `text-[--text-muted]`.
- `{name}` — `text-[--text] font-semibold text-sm`.
- `({client})` — rendered in `text-[--text-muted] text-sm` only when client is non-null.
- `{branch}` — `font-mono text-sm text-[--text-muted]`.
- `phase {paddedPhase}` — `text-sm text-[--text-muted]`.
- `{status}` — `text-sm font-semibold text-[--text]`.
- No second line. No Linear badge. No ADR-touched. No settings link.

Keyboard: `Alt+Left` / browser back is the natural return; the `← All Projects` link provides an explicit in-page target.

### Panel container (shared across all 8 panels)

```html
<section aria-labelledby="{panelId}-title"
  class="rounded-md border border-[--border] bg-[--surface] p-6 flex flex-col gap-4">
  <h2 id="{panelId}-title" class="text-xl font-semibold leading-snug text-[--text]">
    {Panel Title}
  </h2>
  <!-- panel content -->
</section>
```

- Always-expanded: no accordion, no max-height clamp, no disclosure widget (D-4-13).
- `role="status"` added only when the panel is polling (all panels poll at 5s — add `aria-live="polite"` to a visually-hidden `<span>` that announces state transitions, not on the section itself, to avoid announcing every 5s poll).
- Panel titles (Heading 20px/600): CommitmentBlock, Hook Firings, Rationalization Fires, Phase Progress, Execution Timeline, Review Status, Security Status, Verification Status.

## Panel-level Interaction Contracts

### CommitmentBlock (DISC-01)

**Data source:** `GET /api/projects/:id/commitment` → `CommitmentBlockResponseSchema`

**Happy path:** Render the commitment markdown verbatim in a `<pre class="whitespace-pre-wrap font-mono text-sm text-[--text] leading-relaxed">` block. The source filename renders below in 12px `text-[--text-subtle] font-mono` as `Source: {sourceFile}`.

**Empty states:**

| State | Condition | Copy |
|-------|-----------|------|
| No source file | `{ markdown: null, sourceFile: null }` from daemon (no `*.md` in skill-observations) | `No commitment block found. The latest session may not have emitted one yet.` |
| No commitment heading | File exists, no `## Workflow commitment` found | Same copy as above — daemon normalises both to `{ markdown: null, sourceFile: null }` |

**Stale data:** No per-panel stale indicator needed — CommitmentBlock reflects the last-modified `.md` file; its freshness is meaningful over hours, not seconds. The global 5s polling cadence updates it.

### HookFirings (DISC-02 + DISC-04)

**Data source:** `GET /api/projects/:id/observations/recent?limit=20` → `ObservationsRecentResponseSchema`

**Happy path:** Vertical list of up to 20 `HookFiring` rows. Each row:

```
[relative-time]  [skill-bold]  [hook-badge]
```

- Relative time: `text-xs text-[--text-muted] font-mono tabular-nums` (e.g. `3m ago`).
- Skill: `text-sm font-semibold text-[--text]`.
- Hook: `text-xs uppercase tracking-wide rounded px-1.5 py-0.5 bg-[--surface-elevated] text-[--text-muted]` (neutral badge — hooks are informational, not severity-ranked).
- Row separator: `border-b border-[--border]` (no separator on last row).
- List container: `flex flex-col divide-y divide-[--border]` or equivalent.

**DISC-04 install-hint state** (when `skillInstalled: false` from daemon):

```html
<div class="rounded-md border border-[--border] bg-[--surface-elevated] p-4 flex flex-col gap-3">
  <p class="text-sm text-[--text]">
    The meta-observer skill is not installed in this project.
  </p>
  <CodeBlock command="claude skill install meta-observer"
             copyLabel="Copy install command" />
</div>
```

Uses existing `<CodeBlock>` component (Phase 3 re-use). Copy button: `navigator.clipboard.writeText` with sr-only live region feedback (matches existing `CodeBlock` implementation exactly).

**Skill installed, no firings yet:**
`No hook firings yet — try running /review or /cso.`

**Empty states summary:**

| State | Condition | Render |
|-------|-----------|--------|
| Skill not installed | `skillInstalled: false` | Install hint with CodeBlock (D-4-15) |
| Skill installed, no events | `skillInstalled: true`, `entries: []` | `No hook firings yet — try running /review or /cso.` |
| Data present | `entries.length > 0` | Row list (happy path) |

### RationalizationFires (DISC-03)

**Data source:** `GET /api/projects/:id/discipline` → `DisciplineResponseSchema`

**Happy path:** Table (or `<dl>`) of rationalization rows. Each row: label + fire count.

```
Workflow skill not followed       3 fires
Skipped TDD step                  0 fires
```

- Label: `text-sm text-[--text]`.
- Fire count: `text-sm font-mono text-[--text]` for non-zero, `text-[--text-muted]` for zero.
- Zero rows render with `0 fires` (not hidden — zero is a positive signal per D-4-14).
- No fire count cap or truncation. If a rationalization fired 999 times, render `999 fires`.

**Empty states:**

| State | Condition | Copy |
|-------|-----------|------|
| agenticapps-workflow skill not installed | No SKILL.md at allow-listed path | `agentic-apps-workflow skill not installed in this project.` + CodeBlock with install command (placeholder text pending Phase 6 copy tightening per D-4-15 deferred note) |
| Skill installed, empty table | Table parsed but has 0 rows | `No rationalization rows found in SKILL.md.` |
| No fires (normal) | All rows have `fires: 0` | Show table with `0 fires` per row |

### PhaseProgress (PHASE-01)

**Data source:** `GET /api/projects/:id/phase-progress` → `PhaseProgressResponseSchema`

**Happy path:** Checklist of files for the latest phase directory. Canonical file order:

1. `CONTEXT.md`
2. `RESEARCH.md`
3. `UI-SPEC.md` (if exists)
4. `DISCUSSION-LOG.md` (if exists)
5. Per plan (ordered numerically): `04-01-PLAN.md`, `04-01-SUMMARY.md`, `04-02-PLAN.md`, `04-02-SUMMARY.md`, ...
6. `REVIEW.md`
7. `REVIEW-FIX.md` (if exists)
8. `SECURITY.md`
9. `IMPECCABLE.md`
10. `VERIFICATION.md`
11. `HUMAN-UAT.md`

Each file row:

```
[✓ / —]  CONTEXT.md          (2 days ago)
[✓ / —]  RESEARCH.md         (2 days ago)
[—]      UI-SPEC.md
```

- Present file: `CheckCircle2` icon from lucide-react in `text-[--success]` (16px) + filename in `text-sm font-mono text-[--text]` + mtime in `text-xs text-[--text-muted] font-mono`.
- Missing file: `Minus` icon from lucide-react in `text-[--text-subtle]` (16px) + filename in `text-sm font-mono text-[--text-muted]`.
- Filenames render as plain `<code>` text, NOT as links or buttons (`POST /api/projects/:id/open` is deferred per D-4-13 / Phase 5/6 context).
- Phase directory label rendered above the list: `Phase 04 — single-project-view-discipline-phase-progress` in `text-xs text-[--text-muted] font-mono uppercase tracking-wide`.

**Empty state:**
`No phase work yet. Run /gsd-discuss-phase or /gsd-plan-phase to start.`

### ExecutionTimeline (PHASE-02)

**Data source:** Included in `GET /api/projects/:id/phase-progress` → `tdd.timeline[]` in `PhaseProgressResponseSchema`

**Happy path:** Grouped list of TDD task pairs. One group per task ID.

```
Task 04-01
  🔴 test(04-01): add failing tests for commitment parser   2 days ago
  🟢 feat(04-01): implement parseCommitmentBlock            2 days ago

Task 04-02
  🔴 test(04-02): add failing tests for hook JSONL reader   1 day ago
  🟢 feat(04-02): implement readSkillObservations           1 day ago
```

- Task group header: `text-xs font-semibold text-[--text-muted] uppercase tracking-wide` (e.g. `Task 04-01`).
- RED commit row: small circle dot `●` in `text-[--danger]` + commit subject in `font-mono text-sm text-[--text]` + relative timestamp in `text-xs text-[--text-muted] font-mono`.
- GREEN commit row: small circle dot `●` in `text-[--success]` + same pattern.
- Dot: Unicode `●` (U+25CF) rendered as `aria-hidden="true"` with a visually-hidden `<span class="sr-only">` for "RED:" or "GREEN:" prefix on each row for screen reader context.
- Incomplete pair (RED with no matching GREEN): render RED row alone with `text-[--text-muted]` on the subject to indicate pending.
- Tasks ordered by first-commit timestamp ascending (oldest first = natural TDD progression).

**Empty state:**
`No TDD commits yet for this phase.`

| medium | 🟡 | — |

- File status: `text-sm text-[--text-muted]` ("present" / "not run").
- Finding counts: `font-mono text-sm text-[--text]` in an `inline-flex` row with `gap-2`.

**Data source:** `GET /api/projects/:id/security` → `SecurityResponseSchema`

**Happy path:** Two sub-sections:

`No verification run yet — try /gsd-verify-work.`

## State Contracts (all panels)

### Stale data / freshness

All five Phase 4 queries use `staleTime: 5_000` + `refetchInterval: 5_000` + `refetchIntervalInBackground: false` (inherited from D-4-02 / Phase 3 D-02/D-03).

A per-panel freshness warning appears when the most recent successful fetch is older than 15 seconds (3 missed polls) AND a new fetch is in-flight or has errored. This covers "agent restarted and took 8s to come back" without false-positive stale warnings on normal 5s cadence.

Stale-data badge: small pill `Stale` in `bg-[--warning-surface] text-[--warning] text-xs font-semibold rounded px-2 py-0.5` appended to the panel header row (right-aligned via `flex justify-between` on the header). Removed as soon as the query returns fresh data.

**Stale threshold: 15s** (3 × 5s cadence). This is the designer's call — CONTEXT.md left it to discretion.

### Daemon unreachable (per panel)

When a panel's query errors with a network/timeout error (not schema drift):

```html
<div class="flex items-center gap-2 text-sm text-[--warning]">
  <AlertTriangle size={14} aria-hidden="true" />
  Agent unreachable — retrying...
</div>
```

This is an inline row inside the panel (below the panel title), NOT a full-page `DaemonUnreachableState`. The full `DaemonUnreachableState` surfaces at page level via `RepairBanner` on 401. Per-panel unreachable is transient (network hiccup during 5s poll) and should not replace the panel's content — show it alongside the last-known data or the empty state.

### Schema drift (per panel)

Per-panel `<SchemaDriftState>` (existing component) replaces the panel content when the query returns a Zod parse failure. Title for the panel context: `"Schema drift — {PanelTitle}"`. Pattern matches Phase 3 D-07.

### Loading (initial fetch only)

On initial mount (no data yet, `isLoading: true`): render the panel container (title + border + padding) with a single line of `text-sm text-[--text-muted]`: `Loading...`. No skeleton shimmer (Phase 3 D-42 anti-skeleton rule applies — no animated shimmer or placeholder shapes).

## Agent Unreachable — Page-level

If ALL five panel queries error simultaneously (e.g. daemon stopped while user is on the page), the `RepairBanner` triggered by a 401 already surfaces. For non-401 network errors, each panel shows its own per-panel unreachable line (above). The user sees degraded but not blank panels.

No special page-level "all panels unreachable" overlay is rendered in Phase 4. The per-panel inline errors + the 5s retry cadence are sufficient.

## Accessibility Contract

- Every `<section>` panel has `aria-labelledby` pointing to its `<h2>` panel title.
- Every icon is `aria-hidden="true"` with a text label sibling or an `aria-label` on the button.
- Unicode glyphs used as semantic indicators (🔴 etc.) have the glyph `aria-hidden="true"` and the meaning conveyed via `aria-label` on the container (e.g. `aria-label="2 critical, 1 high, 4 medium, 7 low"`).
- Relative timestamps render in a `<time datetime="{iso}">` element where the ISO value is available.
- `⌘C` copy button: existing `CodeBlock` pattern already has `role="status"` sr-only live region for "Copied" feedback.
- Focus rings: `focus-visible:ring-2 focus-visible:ring-[--ring]` on all interactive elements (matches existing pattern throughout the codebase).
- Color is never the sole means of conveying information: RED/GREEN commit pairs have sr-only text prefix (`<span class="sr-only">RED:</span>` / `<span class="sr-only">GREEN:</span>`); severity badges use both glyph + text count.

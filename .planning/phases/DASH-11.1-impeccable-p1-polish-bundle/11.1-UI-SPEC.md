## 1. Domain — What changes

Polish bundle. **Zero new pages.** Four surfaces are modified:

1. **`PageHeader` sticky block** now contains the toolbar (when its host route opts in) — single sticky surface instead of two stacked elements.
2. **`CoverageFamilySection` columns** become identical-width across all family sections via a shared `<colgroup>` and `table-fixed`.
3. **A new `Toast` primitive** surfaces success/error feedback after clipboard writes.
4. **`--color-text-tertiary`** swaps from `#807A92` to `#75708A` (clears WCAG AA on both backgrounds).

The page's information architecture, color story, typography, and interaction model are unchanged. This is hardening + clarity, not redesign.

## 5. Component contracts

### 5.1 `<Toast>` primitive

**Provider:**
```tsx
<ToastProvider>{children}</ToastProvider>
```

**Hook:**
```tsx
const toast = useToast()
toast.show({ message: 'Copied — paste in terminal to index 45 repos', variant: 'success' })
toast.show({ message: 'Copy failed — open the help guide for the command.', variant: 'error' })
```

**Render rules:**
- Position: `fixed top-16 right-4` (below TopBar, right-aligned).
- Z-index: `z-[var(--z-toast)]` (= 2000).
- Width: `min-w-72 max-w-96`; height: content-driven, single line preferred.
- Background: success → `bg-status-success/10` text `text-status-success`; error → `bg-status-error/10` text `text-status-error`.
- Border: 1px solid current foreground at 30% (`border-current/30`).
- Radius: `rounded-md` (8px — matches existing card aesthetic).
- Padding: `px-4 py-2`.
- Shadow: `shadow-card`.
- Glyph (left): `✓` for success, `✕` for error (no Lucide import — text glyph matches the family-aggregate vocabulary).
- Animation: `transition-opacity duration-150 ease-out` only. NO bounce. NO slide. NO scale.
- Lifetime: `duration` prop (default 2400ms). On timeout: fade-out 200ms then unmount.
- Single-slot: rapid second `show()` REPLACES the current toast (no queue).
- ARIA: `role="status"` + `aria-live="polite"` for success; `aria-live="assertive"` for error.
- Keyboard: NO focus-stealing. NO close button v1 (auto-dismiss only).

**API:**
```ts
type ToastVariant = 'success' | 'error'
interface ToastInput {
  message: string
  variant?: ToastVariant  // default 'success'
  duration?: number       // default 2400; 0 = sticky (reserved, unused in 11.1)
}
interface ToastContextValue { show(input: ToastInput): void }
```

## 6. States

### 6.1 Toast states

| State | Visual | ARIA | Trigger |
|---|---|---|---|
| Hidden (initial) | nothing rendered | — | initial |
| Success visible | green tint + ✓ glyph + message | `role=status aria-live=polite` | `show({ variant: 'success' })` |
| Error visible | red tint + ✕ glyph + message | `role=status aria-live=assertive` | `show({ variant: 'error' })` |
| Replacing | new toast fades in over old (no overlap) | aria-live re-fires | rapid 2nd `show()` |
| Dismissing | fade-out 200ms | aria-live remains | duration timeout |

### 6.2 PageHeader sticky states (unchanged from Phase 11; documented for regression contract)

| Scroll position | PageHeader | Family-header | Column-header | Notes |
|---|---|---|---|---|
| 0 | flush at top | flush below | flush below | initial |
| 200/400/800 | sticky flush | sticky flush | sticky flush | stack maintained |
| 1200/1600 | sticky flush | sticky flush | sticky flush | sub-rows still visible |

Regression criterion (P0 — must not regress Phase 11's post-UAT fix): all three rows flush at every scroll position; no transparent gap between any pair.

### 6.3 Column-width invariant

All three `CoverageFamilySection` instances render columns at IDENTICAL pixel widths. Asserted programmatically (`<col>` width-attr equality across 3 sections) and visually (`/browse` screenshot at scrollY 0/400/800 showing column edges align).

## 7. Accessibility

- **Color contrast.** New `--color-text-tertiary` (`#75708A`) clears WCAG AA body text against both `--color-app-bg` (4.69:1) and `--color-sidebar-bg` (4.64:1). Verified by `verify-contrast.test.ts` invariant.
- **Toast announcement.** `role="status"` with `aria-live="polite"` (success) or `aria-live="assertive"` (error). Screen readers announce on every `show()`.
- **Keyboard navigation.** No focus stealing on toast appear. No tabIndex changes. Toast does NOT take focus.
- **Reduced motion.** Toast transition is opacity-only (150ms). Honors `prefers-reduced-motion` automatically because there's no motion to suppress.
- **Touch targets.** No new touch surfaces. Toast does not have a manual-dismiss control v1 (auto-dismiss only) — preserves existing iPad usability.
- **Reader-comfort line-length.** PageHeader subtitle is NOT capped by this phase (P2 #7 deferred). If the planner finds the `max-w-prose` add to be incidental to the toolbar-fold edit, fold in; otherwise defer.

## 8. Anti-features (must NOT happen)

- No new third-party dependencies. Toast is hand-rolled.
- No CSS-Grid refactor of `CoverageFamilySection`.
- No animation extras (bounce, slide, scale).
- No modal-style toast (no backdrop, no focus-trap, no Esc-close v1).
- No multi-toast stack (single-slot only v1).
- No daemon-side code touched.
- No expansion of the text-tertiary token application surface (it inherits automatically; we don't move usages around).
- No mobile-specific UI rework.

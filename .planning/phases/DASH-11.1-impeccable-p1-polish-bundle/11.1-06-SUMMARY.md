# Phase 11.1 Plan 06: Toast Wiring for All 6 Clipboard Call Sites Summary

**One-liner:** AppShellV2 wrapped with ToastProvider; all 6 Coverage clipboard CTAs now fire contextual success/error toasts via useToast() hook with per-site wording.

## What Was Built

- `AppShellV2.tsx` wraps its outermost return in `<ToastProvider>` ‚Äî every authenticated route now inherits the toast context without any call site needing its own provider.
- Six clipboard call sites wired with `async onClick` pattern: await `writeToClipboard()`, then call `toast.show()` with a contextual success message or a generic error message.
- Per-site wording (PD-11.1-02 applied):
  - `IndexGitNexusButton`: "Copied ‚Äî paste in terminal to index your repos with GitNexus"
  - `InstallGitNexusButton`: "Copied ‚Äî paste in terminal to install GitNexus"
  - `CoverageEmptyState` (no-gitnexus): "Copied ‚Äî paste in terminal to install GitNexus"
  - `CoverageFamilySection` (family hint): "Copied ‚Äî paste in terminal to install GitNexus"
  - `CoveragePage` wiki-compile: "Copied ‚Äî paste in terminal to compile the {family} wiki"
  - `CoveragePage` workflow-update: "Copied ‚Äî paste in terminal to update the workflow"
  - All error variants: "Copy failed ‚Äî open the help guide for the command."

**2. [Rule 2 - Missing correctness] CoverageFamilySection withQC helper needed ToastProvider**

**3. [Rule 2 - Missing correctness] CoverageUserJourney needed ToastProvider + clipboard mock**
- **Found during:** Task 6 (GREEN-2c) full coverage suite run
- **Issue:** `CoverageUserJourney.test.tsx` renders `CoveragePage` ‚Äî which now calls `useToast()` ‚Äî without `<ToastProvider>`. Also missing `writeToClipboard` mock needed by the wired components.
- **Fix:** Added `ToastProvider` to wrapper, added `vi.mock('../../../lib/clipboardCompat.js')`.
- **Files modified:** `CoverageUserJourney.test.tsx`

**4. [Rule 1 - Bug] AppShellV2 test: `getAllByRole('status')` needed instead of `getByRole('status')**
- **Found during:** Task 2 (GREEN-1)
- **Issue:** TopBar has an always-present `role="status"` aria-live region (D-6.1-04); `getByRole('status')` threw "Found multiple elements".
- **Fix:** Used `getAllByRole('status')` and filtered by `textContent?.includes('probe')`.
- **Files modified:** `AppShellV2.test.tsx`

**5. [Rule 1 - Bug] Timing test: userEvent + fake timers hung; switched to fireEvent + act**
- **Found during:** Task 3/4 iteration on PD-11.1-05
- **Issue:** `userEvent.setup({ advanceTimers: vi.advanceTimersByTime })` + `mockResolvedValue(true)` hung indefinitely ‚Äî Promise microtasks cannot resolve when the event loop is frozen by fake timers in this configuration.
- **Fix:** Used `fireEvent.click` + `await act(async () => { ... })` which explicitly flushes the microtask queue before advancing timers.
- **Files modified:** `IndexGitNexusButton.test.tsx`

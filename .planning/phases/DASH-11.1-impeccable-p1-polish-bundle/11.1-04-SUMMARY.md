## Hook Design Notes

The `usePageHeaderHeight` hook fires on every `PageHeader` render (not gated by `sticky` prop). This is intentional: simpler implementation with no conditional hook calls; in practice every route mounts exactly one `PageHeader`, so last-writer-wins is never ambiguous. If two `PageHeader` instances ever coexisted in a route, the last one to fire its `ResizeObserver` callback would win. This edge case does not occur in v1.

The `--ph-h` CSS var is published on `document.documentElement` (global scope per PD-11.1-01), not scoped to a route container. Unmount cleanup resets it to `'56px'` (the `:root` default) so the next route's PageHeader starts from the correct first-paint value before its own `ResizeObserver` fires.

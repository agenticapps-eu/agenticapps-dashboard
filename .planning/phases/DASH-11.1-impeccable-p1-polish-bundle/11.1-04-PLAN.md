<objective>
Introduce `usePageHeaderHeight` — a ResizeObserver hook that measures the rendered height of the sticky PageHeader and publishes it as `--ph-h` (a CSS custom property) on `document.documentElement`. This eliminates the hardcoded `top-8` and `top-[5.0625rem]` magic numbers used by `CoverageFamilySection` for its layering-stack sticky offsets — Plan 05 will switch those `top-` values to `calc(var(--ph-h) ± ...)` expressions.

Closes the foundation of P1 #2 from `11-IMPECCABLE.md`. Without this hook, embedding the toolbar inside PageHeader (Plan 05) would silently overlap family-headers because the family-header `top-8` is calibrated for the title-only header height.

Output: 1 new hook file + 1 new test + PageHeader binding + tokens.css default value. Zero visual change on `/coverage` (the published value `--ph-h` is not yet consumed by any element until Plan 05).
</objective>

  <behavior>
    Single-effect hook. ~20 LOC.

    ```ts
    import { useEffect } from 'react'

    export function usePageHeaderHeight(ref: React.RefObject<HTMLElement | null>): void {
      useEffect(() => {
        const el = ref.current
        if (!el || typeof ResizeObserver === 'undefined') return
        const ro = new ResizeObserver((entries) => {
          const entry = entries[0]
          if (!entry) return
          const h = Math.round(entry.contentRect.height)
          document.documentElement.style.setProperty('--ph-h', `${h}px`)
        })
        ro.observe(el)
        return () => ro.disconnect()
      }, [ref])
    }
    ```

    Constraints:
    - NO comments inside the function body
    - NO file-level doc comment beyond one line max
    - Handles missing ResizeObserver gracefully (SSR / very old browser); default `--ph-h: 56px` from tokens.css carries the fallback
  </behavior>

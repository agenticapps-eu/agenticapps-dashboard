### Priority Issues (new — for future phases)

#### [P1] No per-row mutation feedback during gitnexus-analyze dispatch

**Why:** Donald runs gitnexus-analyze on multiple rows. No spinner / disabled state during in-flight mutation. On iPad-over-Tailscale with latency, confidence-killer.

**Fix:** Surface `refresh.isPending` + `{family, repo}` context at the row level. Either spinner on the refresh icon, or toast on settle.

**Command:** `$impeccable clarify`

#### [P1] `<th>` elements have no `title`/tooltip (carried over from Phase 11 P2 #5)

**Why:** "GitNexus" and "Wiki" are proper nouns with specific meaning. First-time visitors and lapsed users cannot recover their meaning without leaving the page.

**Fix:** Add `title=` attributes on each `<th>`, or wire a `TooltipTrigger` primitive with `?` glyph.

**Command:** `$impeccable document`

#### [P2] Wiki column width over-allocates whitespace at current data density

**Why:** `wiki: 'w-[22rem]'` = 352px allocated for the max measured 364px content, but the common case is "never compiled" / "6d ago" cells well under 150px. Creates visual dead space.

**Fix:** Consider `w-60` (240px) with `truncate` + `title=` for full text, OR keep current width with `text-left` content anchoring.

**Command:** `$impeccable layout`

#### [P2] Refresh icon touch target below 44pt Apple HIG minimum on iPad

**Why:** iPad-Donald is a primary persona. 20px effective touch target = ~50% miss rate on fat-finger taps.

**Fix:** `min-w-[44px] min-h-[44px]` wrap on the icon button + relax actions column from `w-8` to `w-10`.

**Command:** `$impeccable layout`

#### [P3] Uncontrolled search input won't re-seed on URL changes

**Why:** `CoverageToolbar.tsx:117` uses `defaultValue={search}` — back-button URL changes don't refresh the input. TanStack Router can preserve component across navigation.

**Fix:** Convert to controlled input (`value={search}`), OR add `key={search}` to force remount.

**Command:** `$impeccable clarify`

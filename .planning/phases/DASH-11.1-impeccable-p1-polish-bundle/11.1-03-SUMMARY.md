## Decisions Made

- **Portal target: `document.body`** (not `#toast-root` dedicated div). Both approaches produce identical test DOM behavior; `document.body` is the idiomatic pattern for singleton feedback surfaces that must escape stacking contexts.
- **`clearTimers()` at both `show()` entry and `useEffect` cleanup.** The `show()` call clears timers before starting the new sequence (replace semantics). The `useEffect` cleanup fires on unmount to prevent React "state update on unmounted component" warnings in test teardown.
- **No `afterEach(vi.useRealTimers)`.** vitest's per-describe timer lifecycle is sufficient; each test starts with `vi.useFakeTimers()` in `beforeEach`.

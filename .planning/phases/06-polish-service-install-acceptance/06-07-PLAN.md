    # Two-Stage Review Protocol

    > Phase 6 POLISH-05. Source: PROJECT.md (non-negotiable: "stages do not collapse â€” they catch different failures").

    ## Why two stages

    Stage 1 catches divergence from the spec / decision log. Stage 2 catches code-quality and security drift. They are run by different skills with different prompts; collapsing them produces the average failure mode of both rather than the best of either.

    ## Stage 1 â€” gstack `/review`

    **Run:** `/review` against the open PR.
    **Reviewer prompt:** spec-compliance against `.planning/PROJECT.md`, `.planning/ROADMAP.md`, the phase's `XX-CONTEXT.md`, and each affected `XX-NN-PLAN.md`.
    **Catches:** locked decision drift (D-XX violations), missing requirement coverage, scope creep, plan-vs-shipped mismatches.
    **Output:** `<finding>` blocks (see schema below) appended to the PR description under `## Stage 1`.

    ## Stage 2 â€” `superpowers:requesting-code-review`

    **Run:** `superpowers:requesting-code-review` against the same PR.
    **Reviewer prompt:** independent code-quality + security pass; reviewer does not read CONTEXT.md (per `~/.claude/projects/.../memory/MEMORY.md` â€” context-blind review surfaces real bugs).
    **Catches:** logic bugs, type-safety regressions, untested branches, security antipatterns, anti-AI-slop drift.
    **Output:** `<finding>` blocks appended under `## Stage 2`.

    ## Sequence

    1. Open the PR with the changeset.
    2. Run Stage 1 -> append findings -> resolve `block`-severity findings before proceeding.
    3. Run Stage 2 -> append findings -> resolve `block`-severity findings.
    4. Merge only when zero unresolved `block` findings remain across both stages.

    Stages run sequentially within the same PR cycle. Do not collapse them into a single review pass.

    ## `<finding>` XML schema

    ```xml
    <finding id="F-001" stage="1" severity="warn">
      <area>Security</area>
      <description>Brief description of what was found</description>
      <evidence>packages/agent/src/routes/registry.ts:263 â€” non-null assertion on re-read</evidence>
      <resolution commit="abc1234">Fixed by adding explicit null guard before returning 404</resolution>
    </finding>
    ```

    | Field | Required | Notes |
    |-------|----------|-------|
    | `id` | yes | F-001, F-002, ... unique per PR |
    | `stage` | yes | "1" or "2" |
    | `severity` | yes | block (stops merge), warn (acknowledged in writing), info (record-only) |
    | `<area>` | yes | Security, Correctness, Performance, A11y, DX, Docs |
    | `<description>` | yes | One sentence |
    | `<evidence>` | yes | file:line OR PR comment URL |
    | `<resolution commit="...">` | when resolved | SHA of the fix commit |

    1. **"Daemon unreachable" inline state.** The daemon is not running on the agent URL the SPA has stored. Run `agentic-dashboard start`. If it was installed as a service, run `launchctl load ~/Library/LaunchAgents/eu.agenticapps.dashboard.plist` (macOS) or `systemctl --user start eu.agenticapps.dashboard` (Linux).

    2. **"Auth token expired" + re-pair banner.** Token rotated either manually or by 30-day auto-rotation. Run `agentic-dashboard pair` to print a fresh pair URL, click it.

    3. **"Schema drift" panel state.** The SPA is running an older bundle than the daemon. Hard-refresh the browser (Cmd+Shift+R / Ctrl+Shift+R). If it persists, the deployed SPA may be behind the daemon â€” check the production deploy.

    4. **LaunchAgent runs but daemon immediately exits.** PATH problem (the most common LaunchAgent failure). Check `~/.agenticapps/dashboard/logs/error.log`. The plist's `EnvironmentVariables.PATH` includes `/opt/homebrew/bin:/usr/local/bin:/usr/bin:/bin` by default â€” if your `node` lives outside those (e.g., NVM), edit the plist to prepend the right path and `launchctl unload ... && launchctl load ...`.

    5. **systemd unit fails to install on older Linux (Ubuntu 18.04 / Debian 9).** The `StandardOutput=append:` directive requires systemd >= 240. Edit `~/.config/systemd/user/eu.agenticapps.dashboard.service` and change `append:` to `file:` (truncates on each restart) for older systems. Then `systemctl --user daemon-reload && systemctl --user restart eu.agenticapps.dashboard`.

    6. **Windows: "command not found".** Windows is not supported in v1 (POLISH-08 deferred). Run on macOS or Linux. WSL2 with systemd-enabled distro works for the Linux path.

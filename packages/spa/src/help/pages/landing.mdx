# AgenticApps

*A spec-first development system for building software with AI collaborators.*

AgenticApps is a workflow, a set of tools, and a small fleet of repositories that together let you ship software in tight collaboration with AI coding agents — without losing the discipline that makes software safe to deploy. It's used in production by [capitalPARX AG](https://capitalparx.com) and a growing list of indie projects.

## What you get

**Discipline.** Every piece of work goes through a brainstorm → plan → execute → verify → review cycle, with explicit gates that catch drift before it compounds. Agents and humans follow the same checklist.

**Multi-host.** The same workflow runs on Claude Code, [pi.dev](https://pi.dev/), and OpenAI Codex CLI. You pick the agent host you prefer; the development discipline stays identical.

**Observable.** Every project produced by the workflow conforms to the same observability contract — structured events, W3C trace IDs, end-to-end correlation across frontend, backend, edge functions, and database. One scan command tells you whether your project conforms; one apply command fixes the gaps with consent.

Three paths in, depending on what you want to do:

### I'm setting up AgenticApps on my machine

Jump to **[Installation](/help/operations/install)**. Ten minutes. Walks you through installing the workflow scaffolder for your chosen host (Claude Code, pi, or Codex), wiring the gates, and running your first phase.

Start with **[Observability Overview](/help/observability/overview)**. Explains what spec §10 requires of every conformant project and how to use the `add-observability` skill to scaffold it in or audit what you have.

## Common questions

**Do I need to use AI agents to use AgenticApps?** No. The workflow is just disciplined software development — the gates and ADRs and two-stage reviews are valuable whether the implementer is human, AI, or both. The AI integration is what makes it *necessary* discipline rather than nice-to-have.

**Is this open source?** Yes — the spec, the workflow scaffolders, the dashboard, and most client templates are MIT-licensed under [github.com/agenticapps-eu](https://github.com/agenticapps-eu).

**Is it production-ready?** The workflow has been used to ship [cparx v0.1.0](https://github.com/agenticapps-eu/cparx) to capitalPARX AG. The dashboard you're using right now is in active development. Treat the workflow as production-grade and the dashboard as a research preview — both improving rapidly.

**Where do I file bugs / suggestions?** GitHub issues on the relevant repo, or `feedback@agenticapps.eu`. For workflow bugs, use [agenticapps-eu/agenticapps-workflow-core](https://github.com/agenticapps-eu/agenticapps-workflow-core) (the spec) — that's the canonical place even if the bug surfaces in a host implementation.
